## 论文主线

这篇论文的核心问题是：**UMI-style robot-free demonstration 很便宜，但不一定 data-efficient（数据高效）**。UMI 让人拿着一个 gripper-like device 去示范，而不是直接 teleoperate 真实机器人；这降低了成本，却带来两个问题。第一，采集者很容易重复录相似的成功轨迹，导致数据集中很多 redundant demonstrations（冗余示范）。第二，wrist camera 视角很局部，遇到遮挡、物体暂时消失或长时程任务进度时，策略可能看不到足够全局状态。

EgoGuide 的回答是把“数据质量”前移到采集时。系统同步记录 wrist image、head / egocentric image、wrist pose、head pose 和 gripper state，然后实时计算当前初始状态是否能扩展已有数据集覆盖。这个 coverage score 会显示在 AR passthrough 里，提示采集者调整手的位置、物体布局、视角或 workspace configuration。采集结束后，系统再用静态规则过滤坏 episode。

![[papers/images/2606.14665v1/overview_page1.png|700]]

**Figure 1: Overview of EgoGuide.** 左侧是 online data curation（在线数据策展）：采集前做 coverage check，采集后做 static quality check。中间是 EgoGuide-UMI 硬件。右侧是 Gated Egocentric Residual Policy（GERP）：wrist camera 生成 base action，ego-camera 生成 residual action，gate 决定是否修正。

论文第二个核心点是 GERP。作者没有让不稳定的 head / egocentric view 直接控制动作，而是把它作为 residual cue（残差信号）。wrist-only policy 是稳定基础，egocentric branch 在 wrist view 遮挡或局部信息不足时才通过 gate 改动作。这个设计非常务实：承认第一视角提供全局上下文，但也承认它会有 viewpoint variation 和 embodiment mismatch。

## Abstract / 摘要讲解

摘要首先指出 real-world demonstrations（真实世界示范）仍是机器人学习的扩展瓶颈。UMI 已经提供了高效的 robot-free collection interface，但当前 UMI-style pipelines 往往会收集 redundant demonstrations，并且缺少 global scene context（全局场景上下文）。

EgoGuide 的两个贡献分别对应这两个瓶颈。

第一，系统层面提出 synchronized wrist and head / egocentric observations（同步腕部与头部/第一视角观测），再加 online visual-geometric data quality guidance。也就是说，系统不仅录数据，还在录制前告诉采集者“这个状态是否有新信息”。

第二，算法层面提出 Gated Egocentric Residual Policy。它不是简单把 wrist image 和 ego image 拼给策略，而是保留 wrist-view policy 作为 stable base，再用 egocentric context 做 gated residual correction。这样可以在遮挡和局部视角不足时利用全局上下文，又避免 head view 的不稳定性破坏基础控制。

摘要中的核心结论是：EgoGuide 能减少所需 data episodes，提高数据效率；GERP 能在 visual occlusion 下进一步增强鲁棒性。

## 1 Introduction / 为什么 UMI 还不够高效

Introduction 首先给出 data efficiency（数据效率）的定义：为了训练到目标成功率，人需要付出多少采集 effort。按照这个定义，UMI 并不天然比 teleoperation 更高效；论文指出 UMI 在一些情况下可能需要超过 5 倍 episodes 才达到 comparable success。

作者定位了两个 bottlenecks（瓶颈）。

第一是 **coverage bottleneck（覆盖瓶颈）**。UMI 数据用 gripper surrogate 收集，再迁移到真实机器人。由于机器人 kinematics、坐标和执行约束不同，策略更需要覆盖多样化状态，而不是大量相似成功轨迹。但是采集者通常不知道哪些状态已经覆盖，哪些状态不足。

第二是 **observability bottleneck（可观测性瓶颈）**。wrist camera 很适合局部控制，但在遮挡、物体离开局部视野或长时程进度判断时不够用。一个只看 wrist view 的策略可能不知道任务整体状态。

EgoGuide 因此把改进分成两层。数据层用 AR-based online guidance 提示采集者去找 novel states。策略层用 GERP 让 head / egocentric context 在需要时修正 wrist-view action。

Introduction 里的重要 claim 有三条：同样数据量下 EgoGuide collection 能显著提高成功率；Pepper Sorting 达到 comparable success 只需 50% demonstrations；egocentric residual policy 能改善 occlusion 下的 performance。

## 2 Related Work / 相关工作坐标

### Robot-Free Demonstration Collection

这一部分把 EgoGuide 放在 UMI 家族里。UMI 是起点，它用便携 gripper-like device 让人类在真实环境里收集可迁移到机器人操作的数据。后续 FastUMI 提升便携性和吞吐，exUMI、TacUMI、ViTaMIn、FreeTacMan 加入 tactile 或 force-related sensing，MV-UMI 和 UMI-3D 增强视觉或 3D 可观测性，DexUMI、HoMMI、HuMI 扩展到 dexterous 或 whole-body manipulation。

EgoGuide 的区别不是再加一个传感器就结束，而是加入 **online guidance**：采集系统会判断当前状态是否补足数据分布。

### Egocentric Vision in UMI Systems

论文提到 ViA、ActiveUMI、ActiveGlasses、EgoMI 等使用 egocentric vision 或 active perception 的路线。这些工作通常关心让机器人学习 active head movement（主动头部运动）或利用移动 ego view。

EgoGuide 做了一个不同取舍：它不把 human head motion 当成机器人要模仿的动作，因为人的头部移动常常是无意的、未受控的。GERP 目标是让策略能部署在 fixed calibrated egocentric camera（固定标定第一视角相机）上，而不是模仿人的头部运动。

### Data Curation and DAgger

DAgger 类方法通过让已训练 policy rollout，再向专家请求纠正来收集 on-policy states。EgoGuide 处理的是更早的 robot-free collection 阶段：此时可靠 policy 可能还不存在，也不希望先上机器人 rollout。因此 EgoGuide 是 training-free data improvement，用采集前 feedback 改善数据，而不是依赖 policy-dependent aggregation。

## 3 EgoGuide System / 系统

### 3.1 EgoGuide-UMI Hardware

![[papers/images/2606.14665v1/hardware_page1.png|700]]

**Figure 2: EgoGuide-UMI hardware and online data curation.** 左侧硬件同步 head image / pose、wrist image / pose 和 gripper state；右侧显示采集前 coverage guidance 和采集后 static filtering。

EgoGuide-UMI 基于 open-source exUMI。硬件包含：

- handheld gripper interface：手持 gripper 记录开合宽度 $g$；
- industrial board-level fisheye camera：安装在 gripper 附近，提供 wrist image $I^W$；
- Meta Quest headset：通过 Unity Passthrough API 提供 head / egocentric image $I^H$ 和 head pose $T^H$；
- Quest controller：安装在 gripper 上，提供 wrist pose $T^W$；
- workstation：保存 episode，计算 coverage score，并把 AR guidance 发回 headset。

同步方式是 UDP。Headset 以 72 Hz 发送 $I^H,T^H,T^W$ 和 recording state；Raspberry Pi controller 以 20 Hz 采集 $I^W$ 和 gripper width $g$，对齐最近 headset timestamp 后发给 workstation。作者报告 cross-modality synchronization 在 20 ms 内，end-to-end workstation latency 在 100 ms 内。

每个 aligned observation 写作：

$$
o = \{I^W, I^H, T^W, T^H, g\}, \quad T^W,T^H\in SE(3).
$$

其中 $I^W$ 是 wrist-view image，$I^H$ 是 head / egocentric image，$T^W$ 和 $T^H$ 是共享 collection world frame 下的 wrist 和 head pose，$g$ 是 gripper width。

### 3.2 Online Data Curation

Online Data Curation 分两步：before recording 的 coverage guidance，以及 after recording 的 static data filtering。

#### Online Data Coverage Guidance

EgoGuide 在每个 episode 录制前估计当前状态是否会扩展数据集覆盖。它使用三类信号：

- wrist image：反映策略输入中的 hand-object appearance 和 contact configuration；
- wrist pose：反映 action-side geometric diversity，因为 UMI 的监督动作和 end-effector motion 相关；
- head / egocentric image：反映物体布局、workspace layout 和 wrist camera 看不到的 global context。

图像 novelty 用 DINOv2 和 CLIP 特征计算。对 view $m\in\{W,H\}$ 和 encoder $e$，先取归一化视觉特征：

$$
z^{m,e}=\bar{\phi}_e(I^m),
$$

再和对应 memory 中最近的 $k$ 个特征做平均 cosine similarity：

$$
s_{m,e}=
\frac{1}{k}\sum_{j\in \mathrm{NN}_k(z^{m,e},M_{m,e})}
(z^{m,e})^\top z_j^{m,e}.
$$

直观含义：如果当前图像和 memory 里最近邻很像，就说明它可能是重复状态；如果相似度低，就说明它可能补足未覆盖区域。作者同时用 DINOv2 和 CLIP，是为了兼顾局部几何/外观和物体/场景语义。

几何 coverage 用 wrist pose $T^W$ 计算。作者把 pose 表示成 translation + quaternion orientation，再用 normalized translation-rotation distance 做 nearest-neighbor pose similarity。

最后，所有 similarity 被转成 novelty percentiles，并显示成三个 0-100 AR guidance values：wrist-view novelty、egocentric-context novelty、wrist-pose novelty。Estimator 以 2 Hz 运行，采集者可以在录制前调整 hand pose、object arrangement、viewpoint 或 workspace layout。

#### Partial Demonstration

普通完整轨迹容易让后段状态重复，比如每次任务后半段都进入类似布局。EgoGuide 允许 demonstrator 从 middle of a task（任务中间）开始录制，把中后期状态变成可控起点。这样可以增加 later-stage interactions 和 recovery behaviors 的覆盖。

#### Static Data Filtering

episode 录完后，系统会在加入 memory 前做 deterministic checks。拒绝条件包括：

- 缺失任一 required modality；
- episode 太短；
- wrist 或 head motion 有 implausible jumps；
- 图像严重 blurred 或 brightness 异常。

Blur 用 grayscale image 的 Laplacian variance 衡量，pose discontinuity 用 linear / angular velocity threshold 检测。作者强调不用 learned filter，是为了避免过滤器引入数据偏置。实测该阶段丢弃约 2% 到 5% episodes。

## 4 Algorithm / Gated Egocentric Residual Policy

![[papers/images/2606.14665v1/algo_page1.png|700]]

**Figure 3: GERP.** 上方 wrist branch 预测 base action，下方 egocentric residual branch 根据 head image 和 wrist pose w.r.t. head 预测 action candidate，再由 gate 融合。

GERP 的基本思想是：wrist-view control 是稳定基础，egocentric view 是可选修正。直接把 moving human viewpoint 作为完整 policy input 很脆，因为 head pose 不是机器人可控动作，head view 可能 noisy 或 redundant，还可能看到手持设备造成 embodiment mismatch。

### Stage 1: Wrist-Only Base Policy

设 demonstration action chunk 为 $A^\star$，按照 UMI convention，它位于 wrist-relative action space。base policy 输入 wrist image、wrist pose 和 language instruction $\ell$，输出 nominal action chunk：

$$
A_b = \pi_b(I^W,T^W,\ell).
$$

这个 base policy 用标准 flow-matching action objective 训练到 $A^\star$，之后作为 stable local manipulation prior。

### Stage 2: Independent Egocentric Residual Policy

第二阶段冻结 base policy，加入 independent residual action expert。为了让 head image 和 wrist pose 坐标一致，作者把 wrist pose 表达到 head frame：

$$
T^{H\leftarrow W} = (T^H)^{-1}T^W.
$$

egocentric branch 输入 head / ego image $I^H$、head-frame wrist pose $T^{H\leftarrow W}$ 和 instruction $\ell$，输出 action candidate $A_r$ 和 scalar gate $\alpha$：

$$
(A_r,\alpha)=\pi_r(I^H,T^{H\leftarrow W},\ell).
$$

最终 action 是：

$$
\hat{A}=(1-\alpha)A_b+\alpha A_r.
$$

这里有一个容易误解的点：$A_r$ 不是直接学习 $A^\star-A_b$ 的残差，而是学习完整 ground-truth action $A^\star$。所谓 residual 是在 fusion 层面：gate 决定最终动作从 base action 朝 egocentric action candidate 移动多少。

训练目标是：

$$
\mathcal{L}_{GERP}
=
\lambda_{res}\mathcal{L}_{FM}(\pi_r;A^\star)
+
\lambda_{act}\left\|(1-\alpha)A_b+\alpha A_r-A^\star\right\|_2^2.
$$

第一项让 egocentric branch 会从 head context 生成动作，第二项训练 gate 组合 base 和 ego candidate。训练使用 curriculum：先只训练 residual branch loss，然后线性增加 composed-action loss，最后保持等权训练。

## 5 Experiments / 实验

### 5.1 Experimental Setup

Robot deployment 使用 Flexiv Rizon 4 robot 和 Grav gripper。测试时，wrist camera 和 Meta Quest controller 以与 EgoGuide-UMI 相同配置安装；egocentric policy 实验中使用固定 Meta Quest headset 作为 head / egocentric camera，位置接近 demonstrator viewing direction。数据采集和机器人评估在不同房间和不同 scene layouts 中完成。

策略实现使用 $\pi_{0.5}$ 作为 base model。图像 resize 到 $224\times224$，动作 horizon $K=16$。policy 训练 30K steps，batch size 128，cosine learning rate 从 $2.5\times10^{-5}$ 到 $2.5\times10^{-6}$。GERP 先训练 wrist-view base policy，再冻结并训练 egocentric branch 30K steps；composed-action loss 在 15K warm-up 后加入，并线性 ramp 10K steps。

指标有两个：

- **Success Rate (SR)**：任务是否完整成功；
- **Task Progress Score (TPS)**：按子目标给 partial completion 分数。

四个任务和 TPS 子目标：

| Task | 子目标 |
| --- | --- |
| Pick Cube | 拿起黄色 cube = 50%；放入 box = 100%。 |
| Pepper Sorting | 拿起 green pepper = 25%；放入 green tray = 50%；拿起 chili pepper = 75%；放入 red tray = 100%。 |
| Garlic Storage | 打开 drawer = 25%；拿起 garlic = 50%；放入 drawer = 75%；关闭 drawer = 100%。 |
| Rubik's Cube | 抓住 cube = 50%；旋转 90 度 = 100%。 |

### 5.2 Comparison on Online Data Curation

主实验比较同样 demo 数量下，unguided UMI collection 和 EgoGuide-guided collection 的效果。所有实验都使用 wrist-only $\pi_{0.5}$ policy，隔离采集质量本身的影响。

| Task | EgoGuide | 100 Demos | 200 Demos | 300 Demos | 400 Demos |
| --- | --- | ---: | ---: | ---: | ---: |
| Pick Cube | no | 25% / 30.0% | 40% / 42.5% | 50% / 55.0% | 70% / 70.0% |
| Pick Cube | yes | 30% / 47.5% | 65% / 75.0% | 95% / 97.5% | 100% / 100.0% |
| Pepper Sorting | no | 0% / 0.0% | 10% / 12.5% | 15% / 45.0% | 50% / 57.5% |
| Pepper Sorting | yes | 0% / 7.5% | 50% / 60.0% | 60% / 60.0% | 75% / 77.5% |
| Garlic Storage | no | 0% / 0.0% | 0% / 0.0% | 0% / 18.8% | 0% / 16.3% |
| Garlic Storage | yes | 0% / 23.8% | 15% / 38.8% | 35% / 53.8% | 50% / 61.3% |

表格里的格式是 SR / TPS。最醒目的结果是 Pepper Sorting：200 demos 时 unguided 只有 10% / 12.5%，EgoGuide 达到 50% / 60.0%。这也是项目页 TL;DR 中的 “10% -> 50%” 和 “2X Speed” claim。

![[papers/images/2606.14665v1/sort_pepper_three_panel_comparison_cropped_page1.png|700]]

**Figure 4: Data scaling comparison.** 这张图按 data size 和 collection time 展示 w/ EgoGuide 与 w/o EgoGuide 的曲线。即使 EgoGuide 每个 sample 有额外引导开销，按总采集时间看仍然更高效。

论文还报告了 distribution evidence。以 CLIP wrist-image features 为例，EgoGuide 覆盖更大的 feature-space region。用 feature covariance trace 衡量，EgoGuide 在两个 camera views 和两个 feature encoders 上分别提升约 5%、4%、3%、4%。这支持“成功率提升来自更好的数据覆盖，而不只是随机波动”的解释。

Partial demonstration 的 Figure 7 显示，从中间状态开始录制可以显著扩大 trajectory distribution，避免后段状态覆盖不足。

### 5.3 Egocentric Residual Policy Evaluation

GERP 对比两个 baseline：

- Wrist Only：默认 UMI 设定，只用 wrist camera；
- Wrist+Ego Direct：把 wrist 和 head / ego images 直接喂给 $\pi_{0.5}$，不做 gated residual 分离。

| Task | #Demos | Wrist Only | Wrist+Ego Direct | GERP |
| --- | ---: | ---: | ---: | ---: |
| Pick Cube | 200 | 65% / 75.0% | 70% / 75.0% | 80% / 90.0% |
| Pepper Sorting | 400 | 75% / 77.5% | 65% / 72.5% | 80% / 87.5% |
| Garlic Storage | 400 | 50% / 61.3% | 50% / 72.5% | 55% / 70.0% |
| Rubik's Cube | 300 | 30% / 37.5% | 70% / 77.5% | 80% / 82.5% |

结果说明：直接加入 ego view 有时会有帮助，如 Rubik's Cube；但也可能伤害性能，如 Pepper Sorting 从 Wrist Only 的 75% 降到 Wrist+Ego Direct 的 65%。GERP 的价值在于它保留 wrist-view base policy 的稳定性，再让 ego view 以 gate 的方式影响动作。

Appendix Table 4 还测试不同固定 egocentric camera placements 下的稳定性：

| Method | Setting 1 | Setting 2 | Setting 3 |
| --- | ---: | ---: | ---: |
| Wrist+Ego Direct | 65% / 72.5% | 60% / 65.0% | 40% / 40.0% |
| GERP | 80% / 87.5% | 75% / 80.0% | 70% / 80.0% |

这支持作者的解释：GERP 对 camera placement 更鲁棒，因为它没有让 ego view 完全接管控制。

## 6 Conclusion / 结论

论文结论总结了两条线。第一，EgoGuide 通过 synchronized wrist/head observations、coverage guidance、partial demonstration 和 static filtering 改善 robot-free demonstration quality。第二，GERP 把 head / ego view 作为 gated residual action cue，解决 wrist camera 局部视角不足，同时保留 wrist-view controller 的稳定性。

用一句话概括：**EgoGuide 不是只追求更多 UMI 数据，而是在采集时主动引导人收集更有用、更不重复的数据，并在策略端保守地利用第一视角上下文。**

### A.3 Training Details

训练 split 在 episode level 做，validation ratio 3%，避免同一轨迹相邻帧进入 train 和 validation。每个 training sample 包含 1 帧 camera observation history、2 帧 low-dimensional input history，以及长度 $K=16$ 的 action chunk。

动作目标包含 relative translation、relative rotation 和 gripper command，并用 training set statistics 做 normalization。训练时 episode 不 downsample，保持 20 Hz。所有实验在 4 张 NVIDIA H100 GPU 和 64-core CPU 上训练；30K-step wrist-only policy 通常约 5 小时，额外 GERP residual policy 约 3 小时。部署评估使用一张 NVIDIA 4070 GPU 和 Intel i9 7900X CPU，policy 运行在 10 Hz。

### D Discussion

作者在 Discussion 中回答了几个关键质疑。

为什么只引导 initial state 能改善整个 episode？因为初始手姿、物体布局和视角会影响后续 approach direction、contact sequence、occlusion pattern 和 recovery behavior。partial demonstration 进一步把中间状态也变成可控初始状态。

partial demonstration 会不会造成 late-stage imbalance？作者承认会增加中后段样本，但认为长时程任务中后段往往更难、普通完整示范后段又容易过于相似，因此更多 late-stage samples 可能帮助训练更平衡的模型。

为什么不用 DAgger？因为 DAgger 需要已有 policy rollout 并请求专家纠正，而 EgoGuide 面向 policy 训练之前的 robot-free data collection 阶段。

为什么不用 object-relative pose？object-relative pose 可能有用，但需要可靠 object tracking / pose estimation；EgoGuide 选择 wrist pose + wrist/head visual features，保持接口简单。

为什么适合 crowdsourced collection？crowdsourcing 容易带来重复、低多样性或 shortcut-like demonstrations。EgoGuide 用 AR novelty score 和 deterministic filtering 让非专家采集更标准化。

### E Limitation / 局限

论文显式列出两条 limitation。

第一是 overhead。EgoGuide 需要额外 feature-memory computation，约 2 秒；还需要采集者调整 scene 和 UMI pose，约 3 秒。正文还报告 real-time guidance 平均每 sample 增加 4.3 秒。不过按总采集时间曲线看，它仍然比 unguided collection 更 data-efficient。

第二是 ergonomics。志愿者戴 AR headset 采集数据会疲劳。作者当前通过短 session 和充分休息缓解，并期待未来换成 AR glasses。

我还会补充三点可追问问题。

第一，实验只在 Flexiv Rizon 4 + Grav gripper 和四个 tabletop tasks 上验证；更复杂接触、双臂、移动操作或真实 crowdsourcing 规模还需要更多证据。

第二，coverage score 依赖 CLIP / DINOv2 features 和 wrist pose distance。它是很实用的 heuristic，但不同任务中“feature novelty”是否总等于“policy useful data”还需要更系统分析。

第三，GERP 的 ego camera 部署为固定视角，而采集时人戴 headset。这个 fixed-ego deployment 设计很稳健，但也意味着它没有解决 active perception 或移动头部控制的问题。

## 和已入库文献的关系

和 [[@lin2026physbrain]] 相比，EgoGuide 不做大规模 VLM brain 训练，而是改善 robot-free demonstrations 的采集质量。PhysBrain 的人类第一视角视频是认知监督来源；EgoGuide 的第一视角观测是 UMI 数据采集和策略鲁棒性的组成部分。

和 [[@tencent2026hy-embodied-05]] 相比，EgoGuide 更底层、更工程化：它关注如何收集可用演示数据，而不是训练一个 embodied foundation model。

和 [[@tang2026frs]] 相比，GERP 的 gate 是多视角 action fusion；FRS 是在 flow policy noise space 中 steering。两者都试图更好地调用已有 policy / action expert，但接口层次不同。

和 [[@kim2026serf]] 相比，EgoGuide 的 ego context 是当前视角补充，不是长期 spatial memory map。SERF 解决长时程状态记忆，EgoGuide 解决采集覆盖和局部视角遮挡。
