## 论文主线

这篇论文的核心问题是：**contact-rich manipulation 的 world model 到底需要什么样的 representation（表征）**。在插入、旋拧、拆卸和探索任务里，机器人不能只识别物体，还要理解接触是否已经发生、摩擦和剪切力怎样变化、目标几何是否被遮挡、以及小误差如何在长时程 rollout 中不断放大。

作者没有把重点放在“提出一个更大模型”，而是构造 ContactWorld 作为 controlled empirical benchmark（受控经验基准）。它固定机器人、控制接口、world-model training 和 planning protocol，然后系统替换视觉模态、触觉模态、fusion strategy、regularization placement 和 predictor architecture，观察什么因素真的影响 latent-space planning。

![[papers/images/2606.13877v1/graph_abstract_page1.png|700]]

**Figure 1 / main finding.** 这张总览图把论文主张压缩成三层：左侧是 12 个 contact-rich tasks，中间是不同视觉/触觉组合的 planning performance landscape，右侧是 long-horizon tactile benefits。关键结论是 PointCloud-only 明显强于 image-only，而 PointCloud+TacFF 是整体最强组合。

论文的主线可以拆成三句话：

1. **空间结构 + 时间连续性最重要。** PointCloud 既保留 3D geometry，又比 wrist image 更少受局部遮挡影响，因此从 20.7% / 22.0% 的 image-only 平均成功率提升到 32.1%。
2. **触觉不是无条件加分。** TacDepth / TacRGB 能帮助 image-based view，但和 PointCloud 不一定兼容；TacFF 因为有 phase-dependent force response，和 PointCloud 更匹配，把整体平均成功率推到 36.1%。
3. **长时程 planning 会放大表征差异。** Goal offset 从 12 增到 48 步时，所有组合都会掉分，但 PointCloud+TacFF 在 48-step 下仍从 PointCloud-only 的 16.0% 提到 20.5%。

## Abstract / 摘要讲解

摘要先提出 contact-rich manipulation 的根本难点：world model 要从 multimodal sensory observations（多模态感知观测）推理 complex contact dynamics（复杂接触动态）。这里的“动态”不只是物体位置变化，还包括被遮挡的接触状态、局部摩擦、剪切力、法向力，以及动作导致的几何约束变化。

ContactWorld 的定位是 benchmark + systematic empirical study。它不只给出一个数据集，而是用 12 个任务和多种模态组合来回答一个更基础的问题：哪些 representation properties 能支持 stable long-horizon planning。

摘要中的主要数值是全文的锚点：

| 设置 | 平均 planning success |
| --- | ---: |
| Wrist View only | 20.7% |
| Front View only | 22.0% |
| PointCloud only | 32.1% |
| PointCloud + TacFF | 36.1% |

这些数值对应两个结论。第一，explicit spatial geometry（显式空间几何）比普通 RGB image 更适合接触预测。第二，tactile sensing 的作用取决于它是否与视觉表征对齐；TacFF 和 PointCloud 都包含空间分布的物理响应，所以更兼容。

摘要最后强调 long-horizon planning。随着 planning objective 变远，预测误差和 contact uncertainty 会累积，触觉的重要性更明显。这一点很关键：触觉的价值不一定在短步目标上立刻显著，但在长 horizon 里能减缓 latent rollout error。

## 1 Introduction / 为什么 contact-rich world model 难

Introduction 把 contact-rich manipulation 和普通视觉主导的操作区分开。普通视觉操作可能主要依赖物体检测、姿态估计和轨迹规划；contact-rich tasks 则要求机器人持续处理几何约束、局部接触事件、摩擦、部分可观测和时间演化反馈。

作者关心的是 latent world models。近年的世界模型常把高维观测编码到 latent state，再预测 future latent dynamics，用于 planning 或 control。JEPA-style latent prediction 尤其适合机器人，因为它可以不重建未来像素，而是直接预测 task-relevant embedding。

但是，接触场景里 latent 表征要满足更苛刻的条件：

- 它必须保留 object / tool / socket 的几何关系，否则插入和拆卸时无法预测下一步接触状态。
- 它必须在接触过程中保持 temporal continuity，否则 rollout 时一个小的观测跳变会让后续预测偏离。
- 它最好直接包含 contact-sensitive signals，因为很多局部接触状态从视觉里不可见。
- 它必须和 planning objective 兼容，因为最终不是做分类或重建，而是优化动作序列。

这也是 ContactWorld 的问题意识：现有工作常比较不同架构、policy 或 benchmark，但较少系统回答“哪个 representation property 对 contact-rich planning 最关键”。本文把这个问题放到统一 benchmark 里检验。

## 2 ContactWorld / 基准设计

### 2.1 Contact-Rich Manipulation Tasks

ContactWorld 包含 12 个任务，分成四类。

| Category | Tasks | 核心难点 |
| --- | --- | --- |
| Insertion | USB, Peg, Power Plug | 狭窄容差、几何对齐、插入阶段目标遮挡。 |
| Disassembly | Spike Barb, Flat Barb, Loose Lid | 摩擦和约束解除、方向性阻力、接触转移。 |
| Screwing | Nut, Bulb, Valve | 持续旋转、长时程接触维护、力和姿态协调。 |
| Exploration | Blind Box / Object Search, Ball Sorting normal/dim | 主动探索、从接触推断隐藏状态或低光照下目标属性。 |

### 2.2 Sensory Modalities and Representation Structure

ContactWorld 的模态设计是全文核心。

![[papers/images/2606.13877v1/modality_vis_page1.png|700]]

**Figure 2 / modality visualization.** USB insertion 过程中，Wrist View 会逐渐看不到 socket；Front View 和 PointCloud 保留更好的全局连续性；TacDepth / TacRGB 变化比较平滑，TacFF 在对齐和插入阶段有明显 force response。

视觉模态：

- **Wrist View**：末端附近 RGB 视角，能看到局部接触细节，但接近目标时容易被夹爪和物体遮挡。
- **Front View**：固定前方 RGB 视角，提供更全局、更连续的场景观察，但几何关系仍是 image-space implicit geometry。
- **PointCloud**：从 front-view RGB-D 重建，形状为 $1024 \times 6$，包含 XYZRGB；它把 3D geometric relationship 直接作为输入。

触觉模态：

- **TacRGB**：触觉传感器内部 RGB image，偏局部外观、纹理和接触区域视觉变化。
- **TacDepth**：触觉表面形变深度图，更直接表达 local contact geometry。
- **TacFF**：tactile force field，形状为 $10 \times 14 \times 3$，每个 taxel 记录 $f_z,f_x,f_y$。它更像一个低分辨率但物理含义明确的空间力场。

Appendix C 的模态规格如下：

| Modality | Type | Shape | Data range |
| --- | --- | --- | --- |
| Wrist View | RGB Image | $256 \times 256 \times 3$ | $[0,1]$ |
| Front View | RGB Image | $256 \times 256 \times 3$ | $[0,1]$ |
| PointCloud | Point Cloud | $1024 \times 6$ | $[-0.5,1.1]$ |
| TacDepth | Depth Map | $320 \times 240$ | $[0,1]$ |
| TacRGB | RGB Image | $320 \times 240 \times 3$ | $[0,0.83]$ |
| TacFF | Force Field | $10 \times 14 \times 3$ | about $[-7e{-3},7e{-3}]$ |
| Action | Control Action | 6 / 7 dims | $[-2,2]$ |

论文用三个高层属性解释这些模态：

| Property | 中文解释 | 哪些模态强 |
| --- | --- | --- |
| Spatial Structure | 是否显式保存几何关系和接触布局 | PointCloud 最强，TacFF 也有 spatial taxel grid |
| Temporal Continuity | 接触演化中是否持续可观测 | Front View / PointCloud 强于 Wrist View，TacFF 在力响应上连续 |
| Contact Sensitivity | 是否直接响应物理接触 | TacRGB / TacDepth / TacFF 强，视觉模态较间接 |

这里有一个重要区分：**contact sensitivity 不等于 planning performance**。触觉都对接触敏感，但如果触觉表示和视觉表示结构不兼容，就可能给 latent dynamics 增加噪声或不稳定变化。

### 2.3 World Model Learning and Planning Evaluation

![[papers/images/2606.13877v1/architecture_page1.png|700]]

**Figure 4 / training and planning pipeline.** 上半部分是 latent world model training：模态编码、latent fusion、action-conditioned prediction 和 regularization。下半部分是 planning：CEM 采样候选动作，用 world model 做 latent rollout，再用 MPC 只执行第一步。

训练阶段采用 JEPA-style latent prediction。给定当前 latent $z_t$ 和动作 $a_t$，GRU-based predictor 预测：

$$
z_{t+1}=P_\theta(z_t,a_t).
$$

主损失是预测 latent 和目标 latent 的 MSE：

$$
L_{\mathrm{pred}}=\|\hat{z}_{t+1}-z_{t+1}\|_2^2.
$$

为了防止 latent collapse，还加入 regularization。默认是 VC regularization（类似 VICReg 的 variance-covariance 约束），并配合 temporal similarity 和 inverse dynamics regularization：

$$
L_{\mathrm{sim}}=\|z_{t+1}-z_t\|_2^2,
$$

$$
L_{\mathrm{idm}}=\|a_t-h_\psi(z_t,z_{t+1})\|_2^2.
$$

总目标写作：

$$
L=L_{\mathrm{pred}}+\lambda_{\mathrm{reg}}L_{\mathrm{reg}}+\lambda_{\mathrm{sim}}L_{\mathrm{sim}}+\lambda_{\mathrm{idm}}L_{\mathrm{idm}}.
$$

规划阶段用 latent-space MPC + CEM。当前观测编码成 $z_t$，目标观测编码成 $z_g$，候选动作序列经过 world model rollout 到 $\hat{z}_{t+H}$，规划 cost 是：

$$
C=\|\hat{z}_{t+H}-z_g\|_2^2.
$$

CEM 每轮选 top candidates 更新采样分布。优化完成后，系统只执行第一步动作，然后下一时刻重新观测、重新规划。这一点很适合 contact-rich tasks，因为接触状态随动作变化快，闭环重规划可以部分修正 rollout error。

训练超参数：AdamW，learning rate $1 \times 10^{-4}$，weight decay $1 \times 10^{-4}$，100k gradient steps，batch size 64；训练 rollout horizon $n=2$。默认 image encoder 是 IMPALA-style CNN，point cloud encoder 是 PointNet-style encoder，TacFF 用 tokenized MLP tactile encoder。

## 3 What Representation Properties Matter / 结果主线

### 3.1 Structured and Continuous Representations Improve Planning

Table 1 的平均行是最重要的结果。

| Vision | Only | +TacDepth | +TacRGB | +TacFF |
| --- | ---: | ---: | ---: | ---: |
| Wrist View | 20.7 | 24.3 | 23.2 | 23.3 |
| Front View | 22.0 | 24.5 | 23.0 | 22.4 |
| PointCloud | 32.1 | 30.2 | 30.7 | 36.1 |

这个表说明：只看视觉时，PointCloud 明显强于 Wrist View 和 Front View。作者的解释不是“点云更高级”，而是点云同时满足两个 property。

第一是 spatial structure。PointCloud 直接保存 object-level 3D geometry，插入和拆卸时能更稳定地表达物体、插孔、约束结构和机器人末端之间的关系。

第二是 temporal continuity。Wrist View 在接近插孔后会因为夹爪和物体遮挡失去目标几何；同一个接触状态可能在图像里看起来模糊或不可见。Front View 稍好，因为它保留全局场景；PointCloud 更进一步，把全局几何变成显式结构。

这也解释了为什么 insertion 和 disassembly 的差异特别明显。这两类任务的成功高度依赖几何对齐和被遮挡接触状态；如果 latent rollout 的输入本身不稳定，后续 planning cost 就很难可靠。

### 3.2 Tactile Effectiveness Depends on Representation Structure

触觉结果更有启发性：触觉不是无条件提升。

对 image-based visual observations，触觉多数有帮助：

- Wrist View 从 20.7% 提到 TacDepth 的 24.3%；
- Front View 从 22.0% 提到 TacDepth 的 24.5%。

直观上，image view 对局部接触不可见或模糊时，触觉能补充 contact cue。

但对 PointCloud，TacDepth 和 TacRGB 反而低于 PointCloud-only：

- PointCloud-only：32.1%；
- PointCloud+TacDepth：30.2%；
- PointCloud+TacRGB：30.7%；
- PointCloud+TacFF：36.1%。

这说明问题不是“要不要 tactile”，而是 tactile representation 和 visual representation 是否 compatible。PointCloud 是空间结构强、几何连续的表征；TacFF 也是空间分布的 force field，而且有 phase-dependent dynamics，因此它提供的是互补的物理动态。TacRGB / TacDepth 虽然接触敏感，但更像 local appearance / local deformation，和 PointCloud 的 geometry latent 不一定自然对齐。

![[papers/images/2606.13877v1/violin_page1.png|700]]

**Figure 5 / tactile performance changes.** 这张图按任务类别看 tactile over vision-only 的变化。TacFF 在 insertion 和 disassembly 中更稳定，TacDepth 在 screwing / exploration 中有时更强，TacRGB 更依赖任务。

这里可以提炼一个建模经验：在 world model 里，tactile modality 应该服务于 action-conditioned dynamics prediction，而不是只追求“传感器信息更多”。如果触觉 latent 的变化方式和视觉 latent 的变化方式不协调，fusion 后的 dynamics 可能更难学。

### 3.3 Long-Horizon Planning Reveals Representation Quality

长时程表是第二个核心证据。

| Offset steps | Wrist only | Front only | PointCloud only | PointCloud+TacFF |
| ---: | ---: | ---: | ---: | ---: |
| 12 | 41.4 | 44.8 | 52.1 | 54.4 |
| 24 | 21.1 | 23.7 | 36.6 | 41.6 |
| 36 | 12.3 | 11.7 | 23.7 | 27.8 |
| 48 | 8.2 | 7.9 | 16.0 | 20.5 |

所有模型都会随着 goal-offset horizon 变长而明显下降。Wrist-only 从 41.4% 掉到 8.2%，Front-only 从 44.8% 掉到 7.9%。这是 contact-rich latent planning 的基本困难：接触状态的微小预测误差会在 autoregressive rollout 中累积，最后规划器优化到一个 latent goal，却无法真实完成几何接触目标。

PointCloud 的优势在长 horizon 下更明显。即使在 48-step，PointCloud-only 仍有 16.0%，约为 Wrist-only 和 Front-only 的两倍。PointCloud+TacFF 又进一步到 20.5%。换句话说，PointCloud 提供长期几何稳定性，TacFF 提供接触动态补充。

![[papers/images/2606.13877v1/multistep_rollout_page1.png|700]]

**Figure 6 / rollout error.** 多步 latent rollout error 随 horizon 单调增加；加入 tactile observations 往往能减缓误差累积。这给 planning result 一个机制解释：触觉并不是只在最终动作上帮忙，而是在 latent dynamics prediction 中改善 temporal consistency。

论文还区分了两个 horizon：

- **Rollout horizon**：world model 自回归预测多少步 latent，是模型预测稳定性的指标。
- **Goal-offset horizon**：当前状态和目标状态之间隔多少环境步，是下游 planning 难度的指标。

这个区分很重要。训练时 rollout horizon 可以较短，但评测时 goal offset 变长，MPC 仍需要反复规划到更远目标；如果 latent dynamics 不稳定，短 rollout 的误差也会在闭环过程中不断显现。

## 4 Conclusion and Limitations / 结论与局限

论文结论可以概括为：contact-rich world modeling 的成败不只取决于模型容量，而强依赖 underlying representation structure。显式空间几何和稳定交互连续性能改善 long-horizon predictive planning；触觉最有效的场景，是它的 representation dynamics 和配对视觉模态兼容。

作者明确列出两个局限。

第一，ContactWorld 的任务虽然多样，但大多仍是 relatively single-stage and goal-directed。真实机器人操作常常需要 hierarchical multi-stage reasoning（层级多阶段推理）、prolonged contact interaction（长时间持续接触）、sequential contact transitions（连续接触转换）和 dynamic replanning（动态重规划）。

第二，虽然论文评估了 goal-offset steps 增大时的鲁棒性，但这些 horizon 相比真实长任务仍然短。实验已经显示 prediction horizon 增加会导致成功率显著下降；实际部署可能需要更强的 long-term temporal consistency、hierarchical planning 和更稳定的 multimodal representations。

我会补充一个从 Appendix G 得出的实践局限：仿真触觉不等于真实触觉。仿真里 TacFF 最强，但真实机器人里 TacDepth / TacFF 需要从原始触觉观测重建，marker tracking、depth estimation、force inference 和 calibration 噪声会明显影响效果。

## Appendix A / 相关工作坐标

### A.1 World Models for Robotic Manipulation

论文把自己放在 latent world models 和 JEPA-style predictive models 的交叉处。早期 World Models、PlaNet、Dreamer 系列证明了 latent dynamics 可以支持 planning 和 policy learning；JEPA 类方法进一步减少像素重建，把重点放到 embedding prediction。

ContactWorld 的差异是：它不主要追求更强架构，而是研究 representation property。作者认为已有 robotic world model 更多比较 architecture、planning objective 或 visual feature，较少系统研究 contact-rich manipulation 中什么表征支撑 robust long-horizon prediction。

这对阅读很有帮助：如果把 ContactWorld 当成新架构论文，会觉得它的模型很轻；但如果把它当成 representation benchmark，就能理解为什么默认 GRU predictor、IMPALA encoder、late concat 反而是合理选择。简单架构降低了 confounder，让模态结构差异更清楚。

### A.2 Vision-Tactile Learning for Contact-Rich Manipulation

触觉方向已有很多工作使用 GelSight、DIGIT 等 vision-based tactile sensors 做 perception、state estimation、imitation learning 或 policy learning。仿真方面 TacSL、DiffTactile、HydroShear 等框架能生成 tactile images / force distributions。

ContactWorld 的切口不是证明“触觉有用”，而是问：在 predictive world model 中，触觉应该以什么 representation 进入？同一个 tactile signal，在 behavior cloning policy 和 latent world model planning 里可能价值不同。后者需要 temporally consistent action-conditioned representations，而不是只需要一帧触觉分类特征。

## Appendix B / 数据集与控制细节

ContactWorld 使用 TacSL，一个基于 Isaac Gym 的 tactile simulation framework。每个任务收集 offline expert demonstrations 和 robot trajectories。数据采集使用 6-DoF 3DConnexion SpaceMouse，人类操作者给 relative end-effector pose delta command，底层 operational space controller 执行。

控制空间：

| Task category | Action dim | Frequency | Control mode |
| --- | ---: | ---: | --- |
| Insertion | 6 | 10 Hz | Relative pose |
| Disassembly | 7 | 10 Hz | Relative pose + gripper |
| Screwing | 7 | 10 Hz | Relative pose + gripper |
| Exploration | 7 | 10 Hz | Relative pose + gripper |

Insertion 默认物体已经被抓住，所以不需要 gripper command；其他任务需要主动夹爪控制，因此是 7 维动作。统一机器人 embodiment 和 action interface 是基准设计的重要部分，因为它让视觉/触觉表征成为主要变量。

## Appendix C / 表征属性分析

Appendix C 强调三类属性不是严格量化指标，而是解释实验趋势的 conceptual abstractions。

**Spatial Structure** 指表征是否保存几何关系。PointCloud 直接编码 3D 点和 RGB 特征，因此最强；TacFF 也以 taxel grid 保存空间分布的力响应。

**Temporal Continuity** 指任务相关信息在交互过程中是否持续可见。Wrist View 在接触中容易遮挡；Front View 和 PointCloud 保持全局连续；TacFF 在持续接触、旋转和力重新分布中有连续变化。

**Contact Sensitivity** 指观测是否直接响应物理接触。所有 tactile modalities 都强于视觉；但高 contact sensitivity 如果没有好的 temporal structure，也未必能改善 planning。

## Appendix D / World Model 架构

默认架构有三个设计取舍。

第一，encoder 不默认用 frozen foundation features。作者用端到端训练的 IMPALA-style CNN 和 PointNet-style encoder，因为 contact-rich world model 需要 action-conditioned interaction dynamics，而不是只需要 semantic invariance。Appendix F.1 的消融也显示 DINOv2 / ViT 表征不一定适合这个目标。

第二，TacFF 单独编码。虽然 TacFF 可以看成低分辨率 2D grid，但它不是普通图像。每个 taxel 是一个三维 force vector。因此作者用 shared MLP 处理 per-taxel force vector，再加可选 positional embeddings 和聚合。

第三，predictor 是 lightweight GRU-based latent transition model。这个选择看起来保守，但对 MPC 很重要：planning 需要大量 rollouts，如果 predictor 很重，在线规划频率会下降。

## Appendix E / Planning and Evaluation Protocol

评测使用 goal-conditioned planning。每个 evaluation episode 选一个当前状态和目标状态，规划器用 latent distance 到目标观测作为 cost。成功标准不是只看物体位置，而是同时看 object pose 和 end-effector pose。

误差项包括：

$$
e^{pos}_{obj}, e^{rot}_{obj}, e^{pos}_{ee}, e^{rot}_{ee}.
$$

成功条件是：

$$
e^{pos}_{obj}<\tau^{pos}_{obj}, \quad e^{rot}_{obj}<\tau^{rot}_{obj},
$$

并且：

$$
e^{pos}_{ee}<\tau^{pos}_{ee}, \quad e^{rot}_{ee}<\tau^{rot}_{ee}.
$$

这个 joint criterion 对 disassembly 和 screwing 很重要，因为物体 pose alone 可能不能说明机器人是否保持正确接触或完成合理交互。

规划超参数：CEM 每步采样 100 条候选 action sequences，迭代 4 次，保留 top-8 candidates；planning rollout horizon 固定为 6；goal-offset horizon 取 12、24、36、48；每个 modality configuration 和 planning horizon 使用 100 个 evaluation trials。

## Appendix F / 额外消融

### F.1 Visual Representation Ablation

作者比较了三类视觉表征：默认 IMPALA-style CNN、frozen DINOv2 features、轻量 ViT。结果显示 IMPALA 更适合主任务，尤其 Front View 设置差距明显。

这不是说 DINOv2 或 ViT “不好”，而是它们的训练目标偏 semantic invariance 和 high-level understanding，未必保留接触预测需要的 action-conditioned temporal dynamics。对 contact-rich world model，能不能预测下一步接触演化比能不能识别物体类别更关键。

### F.2 Tactile Representation Ablation

TacRGB 触觉编码器消融比较了从头训练的 IMPALA tactile encoder、AnyTouch 2 和 Sparsh。对 Wrist / Front image-based world models，预训练 tactile encoders 有轻微帮助，Sparsh 较强。

但对 PointCloud world models，从头训练的 tactile encoder 更强。作者解释为：预训练 tactile encoders 多基于 image-like tactile data，可能更兼容 image visual representations，而不一定兼容 point-cloud geometry。这再次支持 cross-modal compatibility 的主张。

### F.3 Regularization Ablation

VC regularization 在 USB insertion 消融中整体优于 SIGReg。作者认为 contact-rich world modeling 需要保留细粒度 interaction dynamics 和 action-conditioned contact evolution；正则化策略如果过度改变 latent geometry，可能损害这些动态。

### F.4 Cross-Modal Fusion Ablation

作者比较了 Concat、Gate、FiLM 和 Cross-Attention fusion。公式如下：

Late concatenation：

$$
z=[z_{vis};z_{tac}].
$$

Gate fusion：

$$
z=z_{vis}+g(z_{vis},z_{tac})\odot \Delta(z_{tac}).
$$

FiLM fusion：

$$
z=(1+\gamma(z_{tac}))\odot z_{vis}+\beta(z_{tac}).
$$

Cross-attention：

$$
z=\mathrm{Attn}(Q_{vis},K_{tac},V_{tac}).
$$

![[papers/images/2606.13877v1/ablation_fusion_page1.png|700]]

**Figure 11 / fusion ablation.** 复杂 fusion 没有稳定胜过 late concatenation。更重要的是 regularization placement：Concat (Vision-Reg) 让视觉 latent 受 regularization 稳定，而 tactile latent 保留局部接触突变，因此 PointCloud+TacFF 表现最好。

这个结论很实用：多模态模型里，不一定先上 cross-attention。对于触觉这种 local、contact-sensitive、abruptly changing signal，过强的共享正则化或复杂融合可能抹掉它真正有用的变化。

### F.5 Predictor Architecture Study

作者还试了 action-conditioned ViT predictor。它使用 DINO backbone 的 spatial patch tokens、point cloud spatial tokens、tactile condition tokens，并通过 causal attention 做预测。

Table 6 显示，在 USB insertion 上 ViT predictor 整体低于主文 GRU predictor；最高平均是 Wrist+TacRGB 的 19.25%。同时 ViT-based planner 的规划频率约低 20 倍。作者把问题部分归因于 token 数量大，尤其 point cloud 没有 token compression 或 hierarchical aggregation。

这个消融提醒：world model planning 不只看 prediction capacity，还看 online planning throughput。一个强但慢的 predictor 可能在 MPC 里很难用。

![[papers/images/2606.13877v1/time_real_page1.png|700]]

**Figure 12 / real-world valve screwing.** 图中展示真实 rollout 中 Wrist、Front、PointCloud、TacDepth、TacRGB 和 TacFF 随时间的变化。

真实实验支持一个主结论：spatially structured visual representation 仍然强，PointCloud-only 达到 90%。但它也挑战了仿真结论：TacFF 在仿真里是 PointCloud 最强搭档，真实里却没有稳定提升。原因是仿真中的 TacDepth / TacFF 可直接从 physics engine 获得，而真实世界中必须从 raw tactile measurements 重建，容易受 marker tracking、depth estimation、force inference、calibration 和 sim-to-real sensing differences 影响。

这个结果对实际做触觉 world model 很重要。不要直接把“TacFF 最强”理解成“真实触觉力场一定最好”。更准确的读法是：**如果能稳定得到空间分布的 force field，它和 point cloud 很兼容；但真实系统里 force field reconstruction 本身可能是瓶颈。**

## 总结：这篇论文怎么用

如果研究 VLA backbone，ContactWorld 不是首要论文；如果研究 world model / action model / tactile planning，它很值得作为入口。它提供了几个可操作判断：

- 做 contact-rich planning 时，优先考虑显式几何表征，如 point cloud、object-centric state 或其他 spatial tokens。
- 加触觉前先问 compatibility，而不是直接拼模态。TacFF + PointCloud 的互补性来自相似的空间结构和接触动态，不是来自“触觉信息更多”。
- 对 tactile latent 的 regularization 要谨慎。触觉可能需要保留局部突变、剪切和接触转移，过度平滑会伤害预测。
- 对 MPC-style world model，模型吞吐和 planning frequency 是核心指标。大 transformer 如果没有 token compression，可能无法在线规划。
- 仿真触觉结论必须经过真实传感器重建噪声验证，尤其 TacDepth / TacFF 这种 derived representation。

和 [[@kim2026serf]] 的关系：SERF 研究长时程移动操作的 spatiotemporal environment / robot feature map，ContactWorld 研究 contact-rich world model 中何种视觉-触觉表征更稳。两者都在反对 image-only policy，但 SERF 更偏 memory map，ContactWorld 更偏 predictive latent dynamics。

和 [[@lin2026physbrain]] / [[@tencent2026hy-embodied-05]] 的关系：PhysBrain 和 HY-Embodied 关注 VLM/VLA 的 embodied cognition backbone；ContactWorld 关注低层或中层 world-model planning。未来如果把 VLA 与 tactile world model 接起来，ContactWorld 可以提供“触觉进入哪个 latent、如何 regularize、如何评估 long horizon”的设计参考。
