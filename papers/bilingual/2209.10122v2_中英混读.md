# DenseTact 2.0: Optical Tactile Sensor for Shape and Force Reconstruction

## 核心词汇速查

- optical-tactile sensor（光学触觉传感器）：用相机观察软触觉表面的内部图像，再估计接触几何和力。
- calibrated shape reconstruction（标定形状重建）：通过已知几何真值训练/标定模型，使输出深度有物理单位。
- 6-axis wrench estimation（六维力/力矩估计）：同时估计 3 个力分量和 3 个力矩分量。
- randomized pattern（随机化纹理）：印在 gel 表面的不规则点/线图案，用来增强形变可观测性。
- Voronoi stippling（Voronoi 点描）：生成均匀但非规则点分布的方法。
- TSP path（旅行商路径）：把散点连接成连续路径，方便制作可印刷的图案。
- bulging effect（鼓胀效应）：软材料受压时非接触区域也会隆起或变形，影响深度真值。
- ATI force/torque sensor（ATI 力/力矩传感器）：用于采集 6D wrench ground truth 的商用高精度传感器。
- Swin Transformer（Swin 变换器）：基于窗口注意力的视觉模型，本文作为更大模型方案之一。
- transfer learning（迁移学习）：用已有传感器训练好的模型初始化新传感器模型，减少新传感器标定数据。

## 论文主线

DenseTact 2.0 的主线是把 DenseTact 1.0 从“高分辨率 3D 形状重建”推进到“紧凑形态 + 形状重建 + 6D force/torque estimation”。作者做了三件关键事：

1. 重新设计硬件，使传感器更小、更模块化，并提升表面积/体积比。
2. 在 gel 表面加入 randomized pattern，使图像里有更丰富的局部形变线索。
3. 用监督学习同时训练 shape model 和 force model，并用 transfer learning 降低新传感器标定成本。

这篇论文和 DenseTact 1.0 的关系很强：DenseTact 1.0 解决“半球形光学触觉如何恢复 3D surface”；DenseTact 2.0 解决“这个指尖如何更小、更稳定，并同时估计 wrench”。

## 摘要

论文提出 DenseTact 2.0，一种 optical-tactile fingertip sensor。它能从软指尖形变后的内部图像中估计 calibrated shape reconstruction 和 6-axis wrench。作者报告的主要指标是：shape reconstruction accuracy `0.3633 mm per pixel`，force error `0.410 N`，torque error `0.387 N·mm`；transfer learning 可以用约 `12%` 的非迁移训练数据达到相近性能。

![[papers/images/2209.10122v2/main_v3.png|700]]

Fig. 1 是整篇论文的概览图：两个 DenseTact 2.0 sensor 夹住 ATI Nano sensor；每个触觉传感器输出内部图像、depth image 和 3D reconstructed surface；同时模型估计 force/torque，并和 ATI sensor 的真值比较。这个图说明 DenseTact 2.0 的目标是把“几何”和“力”放进同一个 optical tactile pipeline。

## 1. Introduction

作者的应用动机是 collaborative robots（协作机器人）和 dexterous assembly（灵巧装配）。这些任务需要机器人手指既能看见接触形状，又能感知接触力。只知道相机图像或单点压力都不够，因为真实操作中接触区域、接触方向、物体局部形状和力矩会一起变化。

DenseTact 1.0 已经展示了半球形 optical tactile sensor 的 shape reconstruction，但还存在三个需要推进的点：

- 传感器体积仍然偏大，需要更 compact（紧凑）的指尖形态；
- 只重建 shape 不够，还需要 force / torque sensing；
- 每个新传感器都完整采集大数据集的成本太高，需要 transfer learning。

DenseTact 2.0 因此把贡献分成四类：

| 贡献 | 具体做法 | 目标 |
| --- | --- | --- |
| Physical design upgrade | 缩小半球 gel、模块化 camera/LED/gel、镜面内壁 | 更紧凑、更易制造和维护 |
| Patterned gel | Voronoi stippling + TSP continuous path | 增强形变观测，支持 force estimation |
| Calibrated shape + wrench learning | 形状数据和 ATI wrench 数据分别监督 | 同时输出 3D shape 与 6D wrench |
| Transfer learning | 从已有 finger 模型迁移到新 finger | 减少新传感器标定数据 |

## 2. Related Work and Positioning

DenseTact 2.0 的相关工作主要围绕三类传感器：

- GelSight / GelSlim-style sensors：高分辨率几何好，但通常是平面或近似平面设计。
- OmniTact / GelTip / SoftBubble 等 3D optical tactile sensors：能覆盖更大角度，但在 compactness、resolution、force sensing 或标定上各有取舍。
- marker-based tactile sensors：用表面 marker 或 pattern 的移动估计形变和力，更适合 force/flow，但图案设计和大形变鲁棒性是问题。

论文 Table I 的关键比较可以压缩成下面这样：

| 传感器 | 分辨率 | Force | Shape |
| --- | --- | --- | --- |
| DenseTact 1.0 | 800 x 600 | 无 | full |
| DenseTact 2.0 | 1024 x 768 | 约 -11 到 3 N，6-axis wrench | full |

这个表说明 DenseTact 2.0 的核心不是单纯加高分辨率，而是把 force sensing 纳入半球形 optical tactile fingertip。

## 3. Sensor Design

### 3.1 模块化结构

DenseTact 2.0 由 camera module、LED mount、LED strip、gel mount、patterned gel 和 reflective surface 组成。与 DenseTact 1.0 相比，它更小，并且把 gel、照明和相机模块拆得更清楚，便于替换和维护。

![[papers/images/2209.10122v2/explodedview.png|700]]

Fig. 2 展示硬件结构。读这个图时可以注意两点：第一，gel 和 reflective surface 是真正参与接触和形变编码的部分；第二，LED 和镜面内壁改善内部照明，使相机更稳定地观察形变。

论文给出的尺寸是 `32 x 32 x 43 mm`，重量约 `34 g`。与 DenseTact 1.0 相比，接触表面积和体积比从：

$$
\frac{3264.3\ mm^2}{46173\ mm^3}=0.0707\ mm^{-1}
$$

提升到：

$$
\frac{1443.4\ mm^2}{11746\ mm^3}=0.1229\ mm^{-1}
$$

这个指标不是常见的传感器性能指标，但它说明 DenseTact 2.0 在更小体积里保留了较多可接触表面。对 robot fingertip 来说，这比单纯看分辨率更实际。

### 3.2 Gel、图案与反射表面

DenseTact 2.0 的 gel 仍使用与 DenseTact 1.0 类似的软材料，硬度约 `49.7 Shore OO`，半球直径约 `31.5 mm`。为了做 force estimation，作者在 gel 上加入 randomized pattern。

图案生成过程是：

1. 在 `25 mm x 25 mm` 区域中用 Voronoi stippling 生成 `8192` 个点；
2. 用 TSP algorithm 把点连成连续路径；
3. 用 laser-cut stamp plate 和黑色 ink/silicone 混合物把图案转印到 gel 上。

为什么不用规则网格？因为规则图案在大变形下可能出现 aliasing（混叠）和局部对应歧义；随机图案更容易让模型从局部纹理变化中识别形变。

反射表面也被改进。DenseTact 1.0 中反射涂层用 airbrush（喷涂）大约需要 2 小时；DenseTact 2.0 改用 dipping（浸涂），时间约 30 分钟，并且磨损后可以重新浸涂修复。这个改动对实际复现很重要，因为 tactile fingertip 的表面消耗是长期使用中的主要维护问题。

## 4. Data Preparation and Calibration

### 4.1 Shape reconstruction data

DenseTact 2.0 的 shape data collection（形状数据采集）沿用 DenseTact 1.0 的思路：用已知几何的 3D printed indenters 压入传感器，再用 STL model 生成 depth ground truth。但 2.0 做了更自动化的采集，并加入 bulging effect。

bulging effect 指软硅胶受压时，非接触区域也会由于材料不可压缩和超弹性而隆起。DenseTact 1.0 主要把接触物体几何投影成深度；DenseTact 2.0 则把鼓胀区域也纳入生成的 contact shape STL，使真值更像真实软体变形。

形状数据的关键数字：

| 项目 | 数值 |
| --- | ---: |
| Indenters | 21 个 |
| 每次 collection procedure | 8,400 image datapoints |
| Sensor 1 shape train/test | 38,909 / 1,000 |
| Sensor 2 shape train/test | 20,792 / 1,000 |
| Shape dataset size | 8.7 GB / 6.8 GB |
| Normalized depth range | 12.23 mm 到 16.88 mm |
| 每个 depth intensity step | 0.0182 mm |

这里的重点是：2.0 不只是换硬件，还把标定真值做得更贴近软材料实际变形。

### 4.2 Force/torque data

force estimation dataset（力估计数据集）使用 Franka robot arm 随机压入传感器，并用 ATI Gamma SI-65-5 force/torque sensor 提供标签。接触对象包括 cylindrical（圆柱）、spherical（球形）和日常物品/螺母等共 10 类。

![[papers/images/2209.10122v2/force_estimation.png|700]]

Fig. 3 展示力数据采集装置和数据分布。左侧是硬件设置，右侧说明不同 force/torque 维度的数据覆盖范围。这个图的读法是：模型不是从解析力学公式直接推力，而是从大量 image-wrench pairs（图像-力/力矩对）中学习。

数据经过 PSNR filter（峰值信噪比过滤）处理，去除图像质量过差或变化异常的样本。论文报告两个 force sensors 的训练数据量分别约 `38,909` 和 `20,792`，测试集各 `1,000`。

## 5. Model Architecture

DenseTact 2.0 的模型分成 shape estimation model（形状估计模型）、force estimation model（力估计模型）和 transfer learning pipeline（迁移学习管线）。

![[papers/images/2209.10122v2/network_main_v2.png|700]]

Fig. 4 是模型总图。蓝色部分对应 shape estimation，红色部分对应 force estimation，黄色部分对应 transfer learning / preprocessing。整体逻辑是：同一个 tactile image 可以进入不同网络头，分别输出 depth image 或 6D wrench。

论文比较了两类模型：

| 模型路线 | 特点 | 论文中的结论 |
| --- | --- | --- |
| DenseNet + skipped decoder | CNN backbone，参数量较小，类似 DenseTact 1.0 路线 | shape 和 force 结果都更好或相当，训练/推理更快 |
| SwinTF + NeWCRF | transformer-based，更大模型，约 270M 参数 | 参数更多但没有带来更好结果 |

对于 shape reconstruction，模型输入更高分辨率图像，最终处理到 `640 x 640 x 3` 尺度。对于 force estimation，模型直接回归 6D wrench，即：

$$
\mathbf{w} = [F_x,F_y,F_z,\tau_x,\tau_y,\tau_z]
$$

其中 $F_x,F_y,F_z$ 是三个力分量，$\tau_x,\tau_y,\tau_z$ 是三个力矩分量。

## 6. Results

### 6.1 定性结果

![[papers/images/2209.10122v2/performance_v3.png|700]]

Fig. 5 展示 DenseTact 2.0 的 qualitative performance。对 shape reconstruction 来说，可以看 input image、ground truth depth 和 prediction 的对应；对 force estimation 来说，可以看预测 wrench 和 ATI sensor ground truth 的变化是否一致。这个图的作用是说明模型不仅能在简单中心压入时工作，也能处理不同接触位置和形状。

### 6.2 Shape reconstruction

形状重建结果中，DenseNet + skipped decoder 比 SwinTF + NeWCRF 更好。论文报告的关键数值如下：

| 模型 | Sensor 1 L2 test | Sensor 2 L2 test | Total error |
| --- | ---: | ---: | ---: |
| SwinTF + NeWCRF | 0.370 mm | 0.584 mm | 0.4769 mm |
| DenseNet + skipped decoder | 0.282 mm | 0.445 mm | 0.3633 mm |

![[papers/images/2209.10122v2/violin_pos.png|700]]

Fig. 6 展示 position / shape reconstruction error distribution。DenseNet 的优势说明，在这个数据规模和任务上，大 transformer 并没有自动胜出；保持空间细节和稳定训练可能比堆参数更重要。

### 6.3 Force and torque estimation

force / torque estimation 的关键结果如下：

| 模型 | Force error Sensor 3 | Force error Sensor 4 | Torque error Sensor 3 | Torque error Sensor 4 |
| --- | ---: | ---: | ---: | ---: |
| SwinTF | 0.426 N | 0.436 N | 0.416 N·mm | 0.438 N·mm |
| DenseNet | 0.409 N | 0.410 N | 0.395 N·mm | 0.377 N·mm |

作者总结 DenseTact 2.0 的最佳 force error 约 `0.410 N`，torque error 约 `0.387 N·mm`。

![[papers/images/2209.10122v2/violin_force.png|700]]

Fig. 7 是 force / torque error 的 violin plot。这里应关注分布而不只看均值：如果大部分误差集中在低区间，说明模型在常见接触下稳定；如果尾部很长，说明少数接触配置仍然难以估计。

### 6.4 Transfer learning

迁移学习是这篇论文非常实用的一部分。完整采集一个 tactile sensor 的标定数据成本高，因此作者测试了把已有传感器训练好的模型迁移到新 finger。

论文报告：

- 用较小 force dataset 训练 3D force model，normalized wrench absolute mean error 约 `0.05163`；
- 用完整 6D pretrained model 的结果约 `0.05143`；
- 3D force model 收敛更快，约 `7200` steps，而 6D pretrained model 约 `8400` steps；
- position model 也能通过 transfer learning 更快收敛，约 `6900` steps，RMS 约 `0.2275 mm`。

摘要中把这个结果概括为：transfer learning 可以用约 `12%` 的非迁移训练数据获得可比表现。正文 discussion 里也提到约 `10%` 量级的数据。阅读时可把它理解为“只需要原始完整标定数据的一小部分”，具体百分比按实验设置略有差异。

### 6.5 计算开销

论文还比较了模型规模和计算开销：

| 模型 | 参数量 | Forward path |
| --- | ---: | ---: |
| SwinTF route | 约 270M | 约 0.124 s/step |
| DenseNet route | 约 44M | 约 0.04 s/step |

硬件是 NVIDIA 3090 GPU。这个对比说明 DenseNet 不只是准确率更好或相当，也更适合实时触觉管线。transfer learning 训练时间也较可控：论文提到 force 约 `1 h`，position 约 `1.8 h`，使用 A4000、30 epochs、batch size 10。

## 7. Discussion

DenseTact 2.0 的主要价值在于系统工程整合，而不只是某个单点指标：

- 结构更小，能更像机器人手指；
- randomized pattern 增强了图像中的形变线索；
- shape 和 wrench 都被标定到物理量；
- transfer learning 让新传感器校准更接近可用流程；
- DenseNet 路线证明较小 CNN 在触觉图像任务中仍然很强。

但局限也比较明确：

- force output 是 net wrench（合力和合力矩），不是 contact force distribution（接触力分布）；
- 监督学习依赖 ATI sensor 和大量机器人压入数据；
- randomized pattern 的制造一致性、磨损和重印会影响跨批次泛化；
- 模型在 unseen object 或真实 manipulation dynamics 下的长期表现仍需要更多任务级评估；
- 形状和力数据采集流程仍然不算轻量，transfer learning 缓解但没有完全消除标定成本。

作者把 future direction（未来方向）放在 force distribution from single image（从单张图像估计力分布）上。这是合理的：如果 DenseTact 2.0 只能输出合力/合力矩，它对控制已经有价值；但如果能输出空间分布的接触力，就能更直接支持抓取稳定性、滑移预测和局部接触规划。

## 8. Conclusion

DenseTact 2.0 是 DenseTact 系列从“形状重建传感器”向“形状 + 力的紧凑触觉指尖”推进的一步。它的技术路线不是用解析物理模型完全求解软体接触，而是用可制造的硬件、可采集的监督数据和合适的视觉网络，把 tactile image 映射为 depth image 和 6D wrench。

对你读 tactile sensor 文献，这篇可以作为几个比较轴：

- 和 DenseTact 1.0 比：2.0 更小，加入 pattern 和 force/torque estimation；
- 和 DTact / 9DTact 比：DenseTact 2.0 采用反射层和随机图案，DTact 系列更强调材料暗度映射和低成本结构；
- 和 GelSight 比：DenseTact 2.0 更偏 3D curved fingertip，并把 wrench estimation 纳入同一系统；
- 和 3D Cal 类工具比：DenseTact 2.0 仍是 paper-specific pipeline，后续可关注是否能被通用 calibration software 吸收。

## 可追问点

- DenseTact 2.0 的 `0.3633 mm` shape error 和 9DTact/DTact 的 MAE 是否可直接比较？不建议直接比较，因为深度范围、真值生成和表面几何不同。
- randomized pattern 是否一定优于 marker-free？它增强可观测性，但会带来制造和磨损维护问题。
- transfer learning 是否能跨材料批次、跨相机、跨照明设置？论文证明了新 finger 校准有效，但更大范围泛化仍需要验证。
- 如果要用于控制，net wrench 是否足够？对于很多抓取控制可能足够，但对于接触稳定性和局部滑移，force distribution 会更有用。
