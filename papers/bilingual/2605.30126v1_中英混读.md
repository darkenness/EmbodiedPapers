## 论文主线

这篇论文的核心问题不是 reinforcement learning（强化学习），而是 **Large Vision-Language Models（LVLMs，大型视觉-语言模型）如何在不同视觉 token 预算下高效推理**。用户关键词里的 `Elastic Queries` 对应的是本文的 **Conditioned Elastic Queries（条件弹性查询）**：这些 query tokens 不是 RL agent 的查询动作，而是视觉压缩 connector 中可按 budget 截断或扩展的 learnable tokens。

LVLM 的瓶颈来自输入侧：一张图像或一段视频先被 visual encoder 变成几百到几千个 visual tokens，再进入 language decoder。Transformer self-attention 的代价随序列长度近似平方增长，KV-cache 也随 token 数增加。固定压缩方法可以减少 token，但部署前必须选择一个固定 operating point（运行点）：低预算快但丢细节，高预算准但贵。Elastic inference（弹性推理）的目标是 **train once, deploy anywhere（训练一次，多预算部署）**。

![[papers/images/2605.30126v1/compact_efficiency_landscape_page1.png|700]]

**Figure 1: Aggregate retention-efficiency trade-off.** 左图显示 PARCEL 在 16、64、256 visual tokens 下的 mean retention 都高于 MQT 和 M3；右侧表格给出同一 budget 下 image/video 的 theoretical prefill FLOPs 和 LLM KV-cache 成本。直观结论是：如果能在低 token budget 下保留性能，就能把长视频、多图或文档任务的推理成本压下来。

论文认为已有 elastic compression 有两个相反瓶颈。M3 这类 **spatial-only compression（纯空间压缩）** 保留了网格结构，但 successive average pooling 像不完美低通滤波，容易在激进下采样时出现 spectral aliasing（频谱混叠），损伤 chart reasoning、text-centric VQA 等细节敏感任务。MQT 这类 **query-only compression（纯查询压缩）** 使用 non-local learned summaries，弹性很强，但牺牲 explicit grid-aligned tokens（显式网格对齐 token），导致 dense spatial grounding（密集空间定位）变弱。

PARCEL 的回答是 **division of labor（分工）**：让 deterministic spatial pool tokens 做低频 2D layout anchors（布局锚点），让 elastic query tokens 在被 anchors 条件化之后去原始 ViT features 中寻找 complementary visual features（互补视觉特征）。因此它既不完全依赖 pooling，也不完全依赖 query summaries。

![[papers/images/2605.30126v1/PCQR_Teaser_page1.png|700]]

**Figure 4: High-Level Overview of the PARCEL Architecture.** 图中最重要的是 PCQR block：Avg. Pooling 先从 SigLIP features 得到 Anchor Pool Tokens；Matryoshka Query Tokens 先与 anchors 做 Query-Pool Self-Attention，变成 Pool-Aware Query Tokens；随后这些 query tokens cross-attend 到原始 ViT features，输出 Semantic Explorer Query Tokens。最后 pool anchors、semantic explorers 和 text tokens 一起进入 Gemma-2 decoder。

## Abstract / 摘要讲解

摘要的逻辑是：LVLM 把视觉输入映射为 dense token sequences（密集 token 序列），造成 inference（推理）时的 quadratic computational bottleneck（二次计算瓶颈）。Elastic visual-token compression 试图训练一个单一模型，使它能在多个 visual-token budgets 下运行。问题是现有两条路线在 aggressive compression（激进压缩）下各有短板：

- Spatial-only compression（如 nested pooling）像 imperfect low-pass filter（不完美低通滤波器），会产生 spectral aliasing（频谱混叠），遮蔽 fine-grained detail（细粒度细节）。
- Query-only compression（如 nested query resampling）用 non-local summaries（非局部摘要）代替 explicit grid-aligned tokens（显式网格 token），会削弱 spatial grounding（空间落地/定位）。

PARCEL 的核心设计是 **spatial pool tokens + elastic query tokens**。Pool tokens 是低频布局锚点；query tokens 通过 Pool-Conditioned Query Resampling（PCQR）被这些 anchors 条件化，从而避免重复做空间布局编码，把容量留给补充细节。作者在 27 个 benchmarks 上评估，结论是 PARCEL 改善了 performance-efficiency Pareto frontier（性能-效率帕累托前沿），并保持 train once, deploy anywhere。

## 1. Introduction / 为什么需要弹性 visual-token 压缩

Introduction 先指出 LVLM 的输入侧瓶颈。图像、视频、文档或多图输入通常会生成大量 visual tokens；这些 token 进入 language decoder 后，会增加 sequence length，进而提高 self-attention cost、prefill latency 和 KV-cache memory。视觉 token 数量越多，细节越完整，但推理越贵。

现有 static visual token compression（静态视觉 token 压缩）包括 dropping（丢 token）、merging（合并 token）、projection（投影压缩）等。这些方法能降成本，但通常只给一个 fixed-length representation（固定长度表示）。这意味着部署时必须提前选一个点：要么选低成本模型并永久牺牲细节，要么选高分辨率模型并承担高延迟和高内存。

Elastic inference（弹性推理）则希望一个模型支持多个 budgets。本文关注的两个 baseline 是：

- **M3 / Matryoshka Multimodal Models**：通过 successive multi-scale spatial average pooling 构造 nested token structure。优点是保留 2D grid；缺点是激进下采样时容易 aliasing。
- **MQT / Matryoshka Query Transformer**：通过 query transformer + nested dropout 让 query sequence 可截断。优点是灵活；缺点是 query tokens 不是 grid-aligned，空间 grounding 容易变弱。

PARCEL 的 Introduction 把这两个问题称为 representational conflict（表征冲突）：M3 有空间结构但丢高频细节，MQT 有非局部表达但丢空间布局。本文的目标就是把这两种能力拆给不同 token pathway。

## 2. Background and Related Work / 背景与相关工作

这一节的重点不是泛泛介绍 LVLM，而是说明为什么 **matryoshka-style elastic compression（套娃式弹性压缩）** 适合部署但仍不够好。

Matryoshka representation learning 的思想是把可用表示组织成 nested prefixes（嵌套前缀）：小预算用前面一小段，大预算用更长一段，训练时让所有前缀都可用。用在 LVLM connector 上，就可以按 visual-token budget 选择 16、64、256 tokens，而不用为每个预算训练一个单独模型。

但不同 tokenization mechanism（分词/压缩机制）会改变视觉信息的结构：

- Spatial pooling 路线把 $16 \times 16$ feature grid 压成 $8 \times 8$ 或 $4 \times 4$。它天然保留局部邻接和 2D 坐标，但下采样会丢掉超过 Nyquist range（奈奎斯特范围）的高频内容。
- Query resampling 路线用 learnable queries 从视觉特征中抽摘要。它可学习、可截断，但 query token 本身不是图像坐标上的点，难以天然表达 referring expression segmentation（指代表达分割）所需的精细位置。

相关工作还包括 visual token pruning/merging、adaptive inference、efficient multimodal models、PaliGemma/PaliGemma-2 这类开放 LVLM backbone。PARCEL 的位置是 connector-level visual tokenization：它不改 language decoder 的基本形式，而是在视觉特征进入 decoder 之前重新组织 token。

## 3. PARCEL / 方法总览

### 3.1 Spectral Bottlenecks and Spatial Aliasing / 频谱瓶颈与空间混叠

这一节是论文的诊断核心。作者认为 visual-token compression 可以看成一种 downsampling（下采样）。如果下采样前没有充分低通滤波，高频内容会折叠到低频，形成 aliasing（混叠）。对于 LVLM，这些高频内容往往对应小文字、图表线条、边界、细粒度对象属性。

![[papers/images/2605.30126v1/fig_phase2_cdf_64tok_page1.png|700]]

**Figure 3(a): Baseband Concentration (64 Tokens).** PARCEL-Pool 的 cumulative spectral power 在低频处增长更快，说明 pool tokens 更集中承担 low-frequency baseband（低频基带）布局信息。M3 的曲线更宽，表示其 pooling 后的频谱不够干净，和 aliasing / spectral leakage 的解释一致。

![[papers/images/2605.30126v1/fig_phase3_256tok_page1.png|700]]

**Figure 3(b): Spectral Disentanglement (256 Tokens).** PARCEL-Pool 更偏低频，PARCEL-Query 的 footprint 延伸到更高频；这说明 PARCEL 不是让 query tokens 重复编码整体布局，而是让它们补充 pool anchors 之外的信息。MQT 没有这种清晰分工，所以在空间 grounding 上更弱。

Appendix A 给出频谱分析协议。对空间 grid $M \in \mathbb{R}^{H_M \times W_M \times C}$，先做通道内中心化：

$$
\tilde{M}_{h,w,c}=M_{h,w,c}-\frac{1}{H_MW_M}\sum_{h',w'}M_{h',w',c}.
$$

然后对每个通道做 2D DFT（二维离散傅里叶变换）并计算 power spectral density（功率谱密度）。作者使用径向平均 $\bar{P}_Y(r)$ 和归一化径向谱：

$$
\hat{P}_Y(r)=\frac{\bar{P}_Y(r)}{\sum_{k=1}^{r_{\max}(Y)}\bar{P}_Y(k)+\epsilon}.
$$

这里 $\epsilon=10^{-15}$。直观上，$\hat{P}_Y(r)$ 表示频率半径 $r$ 处的相对能量；cumulative spectral concentration 则检查能量是否集中在低频。对 query-based compressor，因为 query tokens 没有原生 2D 结构，作者用 query-to-visual attention weights 形成 attention-weighted footprint，再分析这些 footprint 的频谱。

### 3.2 Pool-Conditioned Query Resampling / 池化条件查询重采样

这一节定义 PARCEL 的 PCQR。设 visual encoder 输出未压缩视觉特征：

$$
X_v \in \mathbb{R}^{N_v \times D}.
$$

根据预算，PARCEL 先做 budget-aware average pooling，得到 grid-aligned spatial anchors：

$$
P \in \mathbb{R}^{N_p \times D}.
$$

并行地，模型维护一组 learnable query tokens $Q_{IN}$。训练时用 nested dropout：采样预算 $B$，在分配 anchors 后只保留前 $N_q=B-N_p$ 个 query tokens，其余丢弃。因为前面的 query 在更多预算下出现，它们会形成 nested prefix structure（嵌套前缀结构），推理时可以直接截断。

关键步骤是先把 query 和 pool anchors 拼接，经过 Query-Pool Self-Attention，得到 Pool-Aware Query Tokens $Q_{PA}$。这一步让 query tokens 在去看原始视觉特征之前先知道 anchors 已经覆盖了哪些 low-frequency layout。随后这些 query tokens cross-attend 到原始 ViT features：

$$
Q_{SE}=\mathrm{CrossAttn}(Q=Q_{PA}, K=X_v, V=X_v).
$$

$Q_{SE}$ 是 Semantic Explorer Query Tokens（语义探索 query tokens），负责抽取 anchors 没有表示的 complementary visual features。最终输入 decoder 的视觉表示是 anchors $P$ 和 semantic explorers $Q_{SE}$ 的拼接。

这套机制的直观意义是：pool anchors 保底空间结构，query tokens 不再从零开始同时学布局和细节，而是在 anchors 的条件下寻找缺失信息。

### 3.3 Budget-Aware Piecewise Routing and Nested Dropout / 分段预算路由

PARCEL 的预算路由规则很简单，但对效果很关键。令 $B$ 是给每张图或每帧视频分配的总 visual-token budget：

$$
B=N_{\mathrm{vis}}=N_p+N_q.
$$

在默认 224x224 输入下，SigLIP visual grid 是 $16 \times 16=256$ tokens。PARCEL 使用两种 anchor grid：

$$
(N_p,N_q)=
\begin{cases}
(16, B-16), & 16 \le B < 64,\\
(64, B-64), & 64 \le B \le 256.
\end{cases}
$$

含义如下：

- **Low budgets (16 <= B < 64)**：使用 $4 \times 4$ anchor grid，即 $N_p=16$。预算极低时至少保留粗空间布局，剩余 token 都给 query。
- **Medium-to-high budgets (64 <= B <= 256)**：使用 $8 \times 8$ anchor grid，即 $N_p=64$。预算变大后，空间 anchors 也变细，剩余 token 再用于探索细节。

Table 3 的消融证明这个路由不是装饰。Fixed 4x4 routing 在 256 tokens 下只有 90.2% retention，因为高预算下 anchors 太粗，query 被迫承担太多中频结构；average pooling-only 虽然有 anchors，但没有 query pathway，16 tokens 只有 83.1%、64 tokens 只有 92.8%。完整 PARCEL 才能在 256/64/16 tokens 同时达到 95.6% / 95.3% / 88.3%。

## 4. Results and Discussions / 实验与讨论

### 4.1 Experimental Setup and Baselines / 实验设置

作者基于 **PaliGemma-2 3B** 实现所有方法。它包含：

- **SigLIP-SO-400M vision encoder**：默认 224x224 输入产生 256 visual tokens。
- **Gemma-2 2B language decoder**：接收视觉 tokens 和文本 tokens，负责生成/回答。
- **Vanilla PG2**：未压缩模型，作为 retention 计算参考。

压缩方法比较三类：

- **M3**：spatial average pooling / spatial-only matryoshka baseline。
- **MQT**：query transformer / query-only matryoshka baseline。
- **PARCEL**：spatial anchors + PCQR query pathway。

Benchmark 共 27 个，覆盖四组：

- **Video understanding**：ActivityNet-CAP、ActivityNet-QA、MSRVTT-CAP、MSRVTT-QA、MSVD-QA。
- **Dense spatial grounding**：RefCOCO、RefCOCO+、RefCOCO-g。
- **Resolution-sensitive tasks**：ChartQA、DocVQA、InfoVQA、SciCap、ST-VQA、TextCaps、TextVQA、WidgetCap 等。
- **General multimodal comprehension**：GQA/xGQA、NLVR2/MARVL、OCR-VQA、AI2D、VQAv2、OKVQA、ScienceQA 等。

### 4.2 Evaluation Protocol / 指标

主指标是 retention（保留率）。对方法 $m$ 和预算 $b$：

$$
\mathrm{Retention}(m,b)=100 \times \frac{s_{m,b}}{s_{\mathrm{PG2}}},
$$

其中 $s_{m,b}$ 是压缩模型分数，$s_{\mathrm{PG2}}$ 是 Vanilla PG2 分数。表中同时报告 raw scores 和 retention；多数主表是 3 random seeds 平均。

### 4.3 Global Efficiency and Pareto Trade-offs / 全局效率-性能前沿

Table 1 是主结论最压缩的版本：

| Modality | Visual Budget | M3 | MQT | PARCEL |
| --- | ---: | ---: | ---: | ---: |
| Image | 256 | 67.6 (91.1%) | 69.1 (93.3%) | 70.4 (95.1%) |
| Image | 64 | 66.5 (89.2%) | 68.1 (91.6%) | 70.1 (94.7%) |
| Image | 16 | 63.9 (85.2%) | 64.3 (85.8%) | 64.8 (86.8%) |
| Video | 256 | 50.6 (92.9%) | 51.4 (94.4%) | 53.1 (98.0%) |

| Video | 16 | 49.8 (91.6%) | 51.2 (94.0%) | 51.6 (95.0%) |

这说明 PARCEL 在 image 和 video aggregate 上都领先，且不是只在高预算领先：16 tokens 的极限预算下也仍然最好。对 deployment（部署）来说，最关键的是低预算仍保留性能，因为 KV-cache 和 decoder prefill 成本直接取决于进入语言模型的 token 数。

| Budget | Image TFLOPs | Video TFLOPs | Image KV | Video KV |

### 4.4 Detailed Benchmark Analysis / 分任务结果

Table 2 把最受压缩影响的任务族拆开。它验证的是第 3 节提出的两个 bottleneck。

#### Video Understanding / 视频理解

Top-3 compression-sensitive video tasks 中，PARCEL 的 mean retention：

- 256 tokens：96.8%，高于 M3 90.1% 和 MQT 91.2%。
- 64 tokens：96.3%，高于 M3 89.7% 和 MQT 89.9%。
- 16 tokens：92.8%，高于 M3 88.8% 和 MQT 91.1%。

这说明多帧输入压缩时，空间 anchors 和 query explorers 的分工也有用。视频任务不只是看静态物体，还要保留 action-relevant temporal evidence（动作相关时间证据）。

#### Image Segmentation / RefCOCO 空间 grounding

RefCOCO suite 最能测试 query-only compression 的弱点。Table 2 的 mean retention：

| Budget | M3 | MQT | PARCEL |
| ---: | ---: | ---: | ---: |
| 256 | 81.7% | 86.4% | 90.6% |
| 64 | 79.6% | 84.9% | 90.8% |
| 16 | 74.2% | 79.0% | 80.5% |

PARCEL 在所有预算都领先，说明 explicit spatial anchors 对 dense localization 很关键。特别是 64 tokens 下 PARCEL 比 MQT 高 5.9 points retention，说明 query tokens 如果没有 anchors，很难自己稳定承担空间布局。

#### Resolution-Sensitive Tasks / 细节敏感任务

DocVQA 和 ChartQA 测试 fine-grained evidence（细粒度证据），包括小文字、表格、图表线条和布局。Top-3 resolution-sensitive tasks 的 mean retention：

| Budget | M3 | MQT | PARCEL |
| ---: | ---: | ---: | ---: |
| 256 | 84.9% | 90.0% | 90.8% |
| 64 | 81.6% | 86.2% | 90.0% |
| 16 | 75.5% | 74.2% | 77.1% |

这里的模式略复杂：M3 在 16 tokens 比 MQT 好一点，因为极低预算下显式 spatial anchors 仍有价值；PARCEL 仍最高，说明 anchor branch 即使没有太多 query tokens，也比单一路径稳。64/256 tokens 时 query pathway 开始补细节，PARCEL 优势更明显。

### 4.5 Ablations on Design Choices / 消融实验

Table 3 是理解方法必要性的关键：

| Ablation | Model Configuration | 256 | 64 | 16 |
| --- | --- | ---: | ---: | ---: |
| Full | PARCEL (Ours) | 95.6% | 95.3% | 88.3% |
| Budget Routing | Average Pooling (4x4) | N/A | N/A | 83.1% |
| Budget Routing | Average Pooling (2x2) | N/A | 92.8% | N/A |
| Budget Routing | PARCEL w/ Fixed 4x4 Routing | 90.2% | 89.6% | 88.3% |
| Budget Routing | PARCEL w/ Fixed 2x2 Routing | 95.6% | 95.2% | N/A |
| Attention Design | ViT-Only Cross-Attention | 95.2% | 95.3% | 87.9% |
| Attention Design | Dual Cross-Attention (ViT + Pool) | 95.4% | 95.2% | 88.0% |
| Baseline Fairness | MQT w/ Self-Attention | 93.3% | 92.5% | 87.8% |
| Baseline Fairness | M3 w/ Self-Attention | 92.2% | 90.4% | 86.8% |

有三点结论：

1. **Budget-aware routing 必须存在**：固定 $4 \times 4$ anchors 在高预算下性能停滞，因为 anchors 太粗；纯 average pooling 又没有 query pathway。
2. **Query 必须 pool-aware**：ViT-only cross-attention 没有先看 anchors，query 容易重复编码低频结构；PCQR 让 query 知道 anchors 覆盖了什么。
3. **不是参数量解释一切**：给 MQT/M3 加 comparable self-attention blocks 后仍低于 PARCEL，说明结构性分工比单纯加容量更重要。

## 5. Conclusion / 结论

论文结论是：现有 elastic compression 在频谱混叠和空间 grounding 之间有 trade-off；PARCEL 通过 spatial anchors + pool-conditioned semantic queries，把低频布局和高频细节拆给两条路径，从而改善 27 benchmark 上的 performance-efficiency Pareto frontier。

作者特别强调 PARCEL 能在 strict 16-token constraints（严格 16 token 约束）下保持 dense recognition、temporal reasoning 和 resolution-sensitive performance。对实际部署，这意味着模型可以根据设备、延迟、输入长度选择 visual-token budget，而不必为每个预算重新训练一个模型。

## Appendix A / 频谱分析协议

Appendix A 解释 Figure 3 的计算方式。对 grid features，作者先中心化、做 2D DFT、计算每个频率位置的 power，然后做径向平均与归一化。对 query-based tokens，因为它们没有 grid 坐标，作者不直接对 query embedding 做 DFT，而是分析 query-to-visual attention footprint：也就是 query 在原始 ViT spatial grid 上关注哪些位置。

这个处理很重要，因为它避免把非空间 query tokens 强行解释成二维图像。作者真正关心的是：query pathway 在视觉空间中“看向”的区域是否覆盖更高频细节。

## Appendix B / 更多实验结果

Appendix B.1 展开所有 PaliGemma-2 benchmarks。总体结论：

- Video understanding：PARCEL 在 256/64/16 tokens 的 mean retention 为 98.0%、97.9%、95.0%。
- Image segmentation：PARCEL 在 RefCOCO suite 上为 90.6%、90.8%、80.5%，所有预算都最高。
- Resolution-sensitive：PARCEL 在 256/64 tokens 最高；16 tokens 下仍有竞争力。
- General vision-language benchmarks：各模型差距较小，因为很多任务已经接近 Vanilla PG2 饱和；PARCEL 在 256/64 tokens 仍略优。

VATEX 被单独报告但不纳入 aggregate，因为 validation split variance 很高且 official test set 不公开。

Appendix B.2 做 448x448 high-resolution stress test。由于计算成本，只报告 single-seed。PARCEL 在 1024/256/64 visual tokens 下的 mean retention 分别为 98.2%、98.2%、95.4%，高于 M3 和 MQT。这说明方法不只在默认 224x224 输入下有效，但高分辨率证据的统计稳健性弱于主实验。

## Appendix C / FLOPs 与 KV-cache 计算

Appendix C 说明 Figure 1 的 theoretical FLOPs 和 KV-cache 估算。默认 224x224 下，SigLIP encoder 每帧输出 $N_v=256$ visual tokens。若每图/每帧压缩预算为 $B$，视频帧数为 $T$，则进入 language decoder 的 compressed visual tokens 数为：

$$
N_{\mathrm{img}}=T B.
$$

加入 text prefix tokens $N_t$ 后：

$$
N_{\mathrm{tot}}=N_{\mathrm{img}}+N_t.
$$

视觉 connector 的成本相对 ViT 和 LLM 很小。PARCEL connector 在 $N_q>0$ 时包含 Query-Pool self-attention 和 Query-to-ViT cross-attention：

$$
C_{QP}=8BD_v^2+4B^2D_v,
$$

$$
C_{Q \to V}=4(N_q+N_v)D_v^2+4N_qN_vD_v,
$$

$$
C_{\mathrm{Q-MLP}}=4N_qD_vM_v.
$$

总 connector cost：

$$
C_{\mathrm{conn}}=T\cdot \mathbf{1}[N_q>0]\left(C_{QP}+C_{Q\to V}+C_{\mathrm{Q-MLP}}\right).
$$

Cross-modal projection cost：

$$
C_{\mathrm{proj}}=2TBD_vD_l.
$$

KV-cache 主要来自 Gemma-2 decoder。对每个 token，key/value cache bytes：

$$
B_{\mathrm{token}}=2 \times 2 \times L_l \times H_{kv} \times d_h.
$$

代入 Gemma-2 常数 $L_l=26$、$H_{kv}=4$、$d_h=256$，得到：

$$
B_{\mathrm{token}}=106{,}496 \text{ bytes}.
$$

因此 visual-token budget 对 KV-cache 有非常直接的影响，尤其是视频输入 $T=16$ 时。

## Appendix D / Benchmark 细节

Appendix D 把 27 个 benchmark 分成几组：

- **Video understanding**：ActivityNet-CAP/QA、MSRVTT-CAP/QA、MSVD-QA、VATEX。关注活动、事件、动作级 temporal reasoning。
- **Dense spatial grounding and segmentation**：RefCOCO、RefCOCO+、RefCOCO-g。要求模型把 referring expressions 对齐到图像区域，直接测试 spatial grounding。
- **Resolution-sensitive document/chart/OCR/screen tasks**：ChartQA、DocVQA、InfoVQA、ST-VQA、TextVQA、OCR-VQA、TextCaps、WidgetCap、Screen2Words、SciCap。关注文本、图表、UI、科学图像和局部细节。
- **General VQA and reasoning**：VQAv2、GQA、xGQA、OKVQA、AOKVQA、AI2D、ScienceQA、TallyQA、CountBenchQA、NLVR2、MARVL、VizWizVQA、RSVQA 等。
- **Captioning and multilingual understanding**：COCO-CAP、NoCaps、COCO-35L、XM3600。

这个 benchmark 设计服务于论文主线：如果 PARCEL 只是 general VQA 稍好，不足以证明其结构价值；真正关键的是它在 RefCOCO、video、DocVQA/ChartQA 这些压缩敏感任务上领先。

## Appendix E / 实现细节

训练时，MQT 的 budget 完全分配给 query tokens；PARCEL 则按 routing rule 把 budget 拆成 anchors 和 queries。在 224x224 设置中：

- MQT 训练时从 $\{2,4,\ldots,256\}$ 采样 query-token budget。
- PARCEL 从 $\{16,18,\ldots,256\}$ 采样总 budget。
- 如果 $16 \le B < 64$，使用 $4 \times 4$ anchors，$N_p=16$，$N_q=B-16$。
- 如果 $64 \le B \le 256$，使用 $8 \times 8$ anchors，$N_p=64$，$N_q=B-64$。

Connector 方面，PARCEL 只用一个 Query-Pool self-attention block 和一个 Query-to-ViT cross-attention block，不重复堆叠。attention hidden width 是 $D_v=1152$，12 heads。

训练流程：

1. Vanilla PaliGemma-2 用 Stage-1 recipe 训练 100M samples。
2. MQT、M3、PARCEL 和 ablations 接入 method-specific connector。
3. 额外做 100M-sample intermediary pretraining；SigLIP vision encoder、Gemma-2 decoder、cross-modal projection 和 connector 都可训练。
4. Transfer tuning 时继续启用 budget sampling；video tasks 的 budget 按每帧应用。

高分辨率 448x448 设置中，SigLIP visual grid 是 $32 \times 32=1024$ tokens。PARCEL 增加第三个 anchor scale：

$$
(N_p,N_q)=
\begin{cases}
(16, B-16), & 16 \le B < 64,\\
(64, B-64), & 64 \le B < 256,\\
(256, B-256), & 256 \le B \le 1024.
\end{cases}
$$

## Appendix F / 局限与社会影响

作者列出的 limitations（局限）主要有三类。

第一，PARCEL 继承 PaliGemma-2 backbone 和 web-scale image-text data 的限制。模型可能反映 demographic（人口统计）、cultural（文化）、geographic（地理）和 linguistic（语言）不平衡。Token compression 本身不会解决这些偏差。

第二，虽然 PARCEL 降低 inference cost，训练和评估 large multimodal models 仍然很贵。论文也承认需要更 compute-efficient training recipes、lightweight ablations 和 low-cost evaluation protocols。

第三，当前 budget list 由 practitioner 指定，而不是从输入自动预测。未来更实用的系统应该有 input-adaptive budget predictor：简单图像用低 budget，复杂文档、图表、视频片段用高 budget。

社会影响方面，降低 visual-token 成本有正面意义：更低内存、更低延迟、更容易在资源受限环境中使用多模态模型，尤其适合 long videos、high-resolution documents 和 multi-image inputs。但也会降低大模型规模化部署门槛，因此同样带来 biased predictions、hallucinated visual interpretations、privacy risks 和 automated decision-making misuse 等风险。

## 与本库研究线索的关系

这篇论文不是 VLA 或 robot policy 论文，但它和具身智能线索有间接关系。真实机器人、多帧视觉、长视频观察、UI/文档工具使用都会遇到 visual-token budget 限制。如果 VLA backbone 需要处理多相机、多帧历史或高分辨率触觉/视觉输入，类似 PARCEL 的 connector 可以作为“预算可控的视觉前端”来参考。

和 [[@tencent2026hy-embodied-05]] 的关系尤其清楚：HY-Embodied-0.5 强调 embodied VLM/VLA backbone 的能力训练；PARCEL 则强调 LVLM visual token interface 的效率和弹性。前者回答“模型脑如何有具身能力”，后者回答“视觉 token 如何在预算下更有效进入语言模型”。

## 可追问点

- 如果把 PARCEL 接到 VLA action expert 前面，spatial pool anchors 是否能帮助 robot spatial grounding，而 query tokens 是否能保留小物体和接触细节？
- 当前 budget 是人工指定的，如何设计 input-adaptive / task-adaptive budget predictor？它能否根据任务类型、图像复杂度、机器人状态或不确定性动态调 token 数？
- 对 tactile images、multi-camera robot observations 或 long-horizon video history，pool anchors 和 query explorers 的分工是否仍成立？
- PARCEL 在 RefCOCO 上证明 spatial anchors 有用，但对 3D grounding、depth、pointing、robot affordance 还没有直接证据。
- 高分辨率实验是 single seed；若扩展到更多 backbone 和任务，Pareto frontier 是否稳定？
