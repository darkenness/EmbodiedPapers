## 论文主线

这篇论文要回答的问题是：当双臂任务越来越紧耦合时，仅靠大容量 VLA 端到端学出隐式协作是否足够？作者认为不够。现有 VLA 通常把左右臂 joint commands（关节命令）拼成 14 维 monolithic vector（单体向量）直接回归，这会把“共同任务意图”和“各臂执行细节”混在同一表征里，导致同步强度、角色不对称、运动平滑性和安全约束都难以解释，也更难在部署时调节。

![[papers/images/2606.20285v1/system_overview_0303.png|700]]

**Fig. 1: Overview of our dual-arm VLA system.** 系统总览：VLM backbone 产生 hidden states，SAE 把它们解释成 structured latent space（结构化潜空间），再经 separate projection heads（独立投影头）输出分解动作；部署时 LAC 读取 shared/residual 结构，对 joint-level commands 做 refinement（精炼），目标是更准确、更同步、更平滑、更安全。

Co-VLA 的三层设计彼此分工：

1. **SAE（学的时候结构化）**：在 $\pi_0$ 骨干上替换 monolithic action head，把动作拆成 shared + residual，并配合 coordination-aware auxiliary losses（协调感知辅助损失）按任务模式塑形。
2. **LAC（部署的时候可解释地调节）**：不改策略权重，只根据 shared/residual 能量与方向关系，自适应低通滤波和平滑 joint commands。
3. **Co-Motion（数据层面的探索）**：在 RoboTwin 2.0 代码生成管线里让双臂更多并发运动，提高协调样本密度，但也带来 efficiency-vs-learnability trade-off（效率与可学习性权衡）。

实验主线分四段：RoboTwin 2.0 上 Co-VLA 相对 $\pi_0$ / $\pi_0.5$ 提升平均 Easy 成功率；Co-Motion 缩短示范采集时间 10–25%，但训练更难；真实 AgileX Cobot Magic 上 SAE+LAC 在 ID/OOD 多数设置优于 $\pi_0$；latent–behavior alignment（潜变量-行为对齐）和 auxiliary loss / LAC 消融说明结构确实对应可解释协作行为。

## 摘要与核心贡献

摘要的逻辑链条是：VLA 在单臂和双臂操作上都表现很强，端到端学习甚至能让 coordinated bimanual behaviors（协调双臂行为）涌现出来；但当双臂任务更 tightly coupled（紧耦合）、执行约束更关键时，implicit coordination alone（仅靠隐式协调）不足以保证 reliable、interpretable、deployment-stable（可靠、可解释、部署稳定）的行为。

Co-VLA 的回应是在 action generation level（动作生成层）引入 explicit structural priors（显式结构先验）：

- 用 **SAE** 替换 monolithic action head；
- 用 **modular coordination-aware loss** 按任务结构塑形 shared / residual latents；
- 用 **LAC** 在部署时调节 synchronization strength（同步强度）、execution asymmetry（执行不对称）、motion smoothness（运动平滑）和 safety constraints（安全约束）。

论文报告的三组关键数字：

- tight-coordination tasks 上相对 monolithic baseline **+27%** success rate；
- OOD real-world 从 **13% → 27%**，超过翻倍；
- task completion time 最多减少 **25%**。

## I. Introduction / 研究问题

Introduction 先肯定 VLA 在单臂和双臂上的进展：RT-2、OpenVLA、$\pi_0$、ALOHA 等说明大视觉-语言骨干 + continuous action prediction（连续动作预测）确实能学出协作行为，有时甚至不需要 explicit collaboration modeling（显式协作建模）。

但作者指出三类越来越难被隐式消化的需求：

1. **temporal synchronization（时间同步）**：双臂要同时加速/减速，而不是各走各的。
2. **asymmetric role assignment（不对称角色分工）**：一臂固定、一臂操作，或一臂递送、一臂接收。
3. **execution-level constraints（执行级约束）**：避碰、轨迹平滑、安全中间姿态等。

现有 VLA 的问题是：左右臂动作都从 shared representation（共享表征）直接回归，**task-level coordination intent** 与 **residual execution details** 没有分开。于是协作强度、不对称性、安全行为都埋在黑盒里，失败时也难诊断。

本文的核心观察可以概括成一句：**collaboration in bimanual manipulation is not an action itself, but a structure over actions（双臂协作不是一个动作，而是动作之上的结构）**。两臂常常共享“共同搬运/共同交接”的任务意图，但在 joint level 需要不同执行策略。把双臂动作压成 monolithic vector，会把两种来源的 variation（变化）混在一起，伤害泛化，也让协作失败难以定位。

因此 Co-VLA 引入三块：

1. **SAE**：shared latent 表同步意图，residual latents 表各臂执行细节。
2. **LAC**：部署时解释这些 latent，调节同步、平滑、安全。
3. **Co-Motion**：数据采集时让双臂更多并发运动，补 coverage。

## II. Related Work / 相关工作与定位

### VLA for Robotic Manipulation

相关工作覆盖 RT-2、OpenVLA、Octo、$\pi_0$、$\pi_0.5$、Diffusion Policy、RDT-1B 等。它们证明 coordinated dual-arm behavior 可以涌现，但 action head 仍投影到 monolithic vectors，没有分离 coordination intent 与 arm-specific execution。

**本文定位**：保留 $\pi_0$ backbone，只在 action head 用 SAE 做 structured decomposition（结构化分解）。

### Dual-Arm Coordination

经典双臂控制有 operational space formulations（操作空间公式）、virtual linkage model（虚拟连杆模型）、hybrid position/force coordination（位置/力混合协调）、impedance-based methods（阻抗法）等，多在 control level 解析施加协调。

**本文差异**：在 representation level 加结构，仍兼容 standard joint-level pipelines（标准关节级控制管线）。

### Structured Action Representation

shared–private decomposition（共享-私有分解）见于 multi-task learning、QMIX、MAPPO 等；IACE 为双臂同步提出 inter-arm coordinated encoders。但这些思想尚未进入 VLA action heads。

**本文差异**：首次把 shared–residual 结构嵌进 VLA action expert，并配合部署时 LAC。

### Demonstration Paradigms for Bimanual Learning

ALOHA / Mobile ALOHA / ALOHA Unleashed 用 puppeteering teleoperation（傀儡式遥操作）采集双臂示范；RoboTwin / RoboTwin 2.0 用 LLM 代码合成脚本示范。常见问题是 arm motions 往往 sequential or loosely coupled（顺序或弱耦合），underrepresenting temporal coordination（低估时间协调）。

**Co-Motion 定位**：补 concurrent coordination samples，但不是方法生效的必要条件。

### Execution Refinement

residual policy learning、control barrier functions、impedance controllers 等可在执行层 refine policy。阻抗控制通常需要 force sensing（力传感）。

**LAC 定位**：joint-command level refinement，解释 SAE 的 shared/residual 语义，不修改 backbone，也不要求力控硬件。

## III. Method / 方法

### III-A Structured Action Expert (SAE)

#### 标准 VLA 的非结构化动作生成

在 $\pi_0$ 这类 VLA 中，action head 把 transformer hidden state $h_t \in \mathbb{R}^H$ 直接映射到双臂动作：

$$a_t = f_{\pi_0}(h_t), \quad a_t = [a_{t,L}, a_{t,R}] \in \mathbb{R}^{14}$$

这里 $a_{t,L}, a_{t,R} \in \mathbb{R}^7$ 是左右臂 joint velocities（关节速度）。这种写法把双臂动作当成 monolithic vector，inter-arm coordination 只能隐式学习。

![[papers/images/2606.20285v1/co-vla-SAE-new.png|700]]

**Fig. 2: Structured Action Expert (SAE) architecture.** SAE 用 parallel latent decoders（并行潜变量解码器）替换单一投影头：同一 $h_t$ 同时产生 shared latent 与左右 residual latent，再分别投影成 shared action components 和 residual action components，最后相加得到各臂命令。

#### Shared–Residual Action Decomposition（共享-残差动作分解）

给定 $h_t$，SAE 计算三个 latent：

$$z_t^s = g_s(h_t), \quad z_t^L = g_L(h_t), \quad z_t^R = g_R(h_t)$$

- $z_t^s \in \mathbb{R}^L$：shared coordination intent（共享协调意图）
- $z_t^L, z_t^R \in \mathbb{R}^L$：各臂 execution-specific information（执行特异信息）

shared latent 通过左右各自的 shared projection heads 产生共同动作分量：

$$a_{t,L}^s = \phi_L^s(z_t^s), \quad a_{t,R}^s = \phi_R^s(z_t^s)$$

residual latents 通过 residual heads 产生微调分量：

$$a_{t,L}^r = \phi_L^r(z_t^L), \quad a_{t,R}^r = \phi_R^r(z_t^R)$$

最终 joint commands 用 additive composition：

$$a_{t,L} = a_{t,L}^s + a_{t,L}^r, \quad a_{t,R} = a_{t,R}^s + a_{t,R}^r$$

直观理解：

- $a^s$：macro motion / task-level collaboration（宏观协作运动）
- $a^r$：micro adjustment / arm-specific deviation（微观微调）

接口层面仍输出 14 维 joint velocities，因此可以接到现有 $\pi_0$ 训练与部署栈上。

#### Task-Adaptive Coordination Losses（任务自适应协调损失）

仅有结构分解还不够，不同双臂任务属于不同 coordination regimes（协调模式）。作者设计三种可切换的 auxiliary losses：

**1. Sparse Residual Regularization（稀疏残差正则）** — 适合 near-symmetric execution（近对称执行）

$$\mathcal{L}_{\text{sparse}} = \mathbb{E}_t\left[\|a_{t,L}^r\|_1 + \|a_{t,R}^r\|_1\right] \tag{1}$$

鼓励 common motion 进入 shared latent，只在必要时激活 residual。

**2. Shared Mean Velocity Consistency（共享均值速度一致性）** — 适合 asymmetric role assignment（不对称角色分工）

$$\mathcal{L}_{\text{shared}} = \mathbb{E}_t\left[\|a_{t,L}^s - \bar{u}_t\|_2^2 + \|a_{t,R}^s - \bar{u}_t\|_2^2\right], \quad \bar{u}_t = \tfrac{1}{2}(u_{t,L}+u_{t,R}) \tag{2}$$

让 shared component 对齐双臂平均速度趋势，强化“shared latent = coordination intent”的语义。

**3. Temporal Synchronization Loss（时间同步损失）** — 适合 temporally coupled motion（时间耦合运动）

定义速度变化幅度 $m_{t,L}=\|\Delta a_{t,L}\|_2$、$m_{t,R}=\|\Delta a_{t,R}\|_2$，标准化后算时间耦合 $\mathrm{corr}_{\text{pred}}=\mathbb{E}_t[\tilde{m}_{t,L}\tilde{m}_{t,R}]$，再：

$$\mathcal{L}_{\text{sync}} = 1 - \mathrm{corr}_{\text{pred}} \tag{3}$$

鼓励两臂加速/减速节奏同步，而不强制方向相同。

总目标：

$$\mathcal{L} = \mathcal{L}_{\text{action}} + \lambda \mathcal{L}_{\text{aux}}, \quad \mathcal{L}_{\text{aux}} \in \{\mathcal{L}_{\text{sparse}}, \mathcal{L}_{\text{shared}}, \mathcal{L}_{\text{sync}}\} \tag{4}$$

其中 $\mathcal{L}_{\text{action}}$ 是原 VLA 动作监督损失；$\lambda=0.001$。损失按任务 dominant coordination pattern（主导协调模式）人工选择，实验节用 Table VI 验证这种 task-level selection 合理。

### III-B Latent-Aware Controller (LAC)

SAE 已经让预测动作带 shared/residual 结构，但直接执行仍可能出现：

- high-frequency jitter（高频抖动）
- overreaction to small residual signals（对小 residual 过度反应）
- insufficient protection of precision-critical motions（精密微调保护不足）

LAC 在 deployment time（部署时）读取 SAE 分解结果，对 raw joint command chunk $\{q_t\}_{t=1}^H$ 做 refinement，输出 $\{\tilde{q}_t\}$。它**不新增学习模块，也不改训练好的 policy**。

#### Energy-Based Micro–Macro Analysis（基于能量的宏-微分析）

每步计算 shared / residual 平均能量：

$$E_t^s = \tfrac{1}{2}(\|a_{t,L}^s\|_2 + \|a_{t,R}^s\|_2), \quad E_t^r = \tfrac{1}{2}(\|a_{t,L}^r\|_2 + \|a_{t,R}^r\|_2)$$

micro-motion ratio：

$$\rho_t = \frac{E_t^r}{E_t^s + \varepsilon}$$

$\rho_t$ 大说明当前步 residual 更重要。

#### Residual Opposition and Collaboration Signal（残差对立与协作信号）

计算 residual 余弦相似度：

$$\cos_t = \frac{\langle a_{t,L}^r, a_{t,R}^r \rangle}{\|a_{t,L}^r\|_2 \|a_{t,R}^r\|_2 + \varepsilon}, \quad \omega_t = -\cos_t$$

opposition score $\omega_t$ 高，表示两臂 residual 在相反方向施力，常见于 stabilizing / holding / fine alignment（稳定、夹持、精细对准）。低 opposition 更像 jitter。

#### Adaptive Stiffness Modulation（自适应刚度调节）

围绕 base stiffness $\alpha_{\mathrm{base}}$，LAC 分三种 regime：

| 模式 | 条件 | 刚度 | 含义 |
| --- | --- | --- | --- |
| Macro-dominant | $\rho_t < \tau_\rho$ | $\hat{\alpha}_t = \alpha_{\mathrm{base}} + \Delta_{\mathrm{macro}}$ | 共享运动主导，允许更快更直接执行 |
| Precision-critical | $\rho_t \ge \tau_\rho \land \omega_t > \tau_\omega$ | $\hat{\alpha}_t = \alpha_{\mathrm{base}} + \Delta_{\mathrm{prec}}$ | 保护有意义的双向微调 |
| Noise suppression | 其他 | $\hat{\alpha}_t = \alpha_{\mathrm{base}} - \Delta_{\mathrm{noise}}$ | 抑制不协调高频 residual |

然后 clip 到 $[\alpha_{\min}, \alpha_{\max}]$，并做 temporal smoothing：

$$\alpha_t = (1-\beta)\alpha_{t-1} + \beta \hat{\alpha}_t$$

#### Joint-Level Action Refinement（关节级动作精炼）

对原始命令 $q_t=[a_{t,L};a_{t,R}]$ 做一阶低通：

$$\tilde{q}_t = (1-\alpha_t)\tilde{q}_{t-1} + \alpha_t q_t$$

$\tilde{q}_t$ 才是发给机器人的 refined command。LAC 本质是 **macro–micro control strategy（宏-微控制策略）**：shared 高保真传递，residual 按语义选择性保护或抑制。

### III-C Co-Motion: Collaborative Motion Paradigm

标准双臂示范常常是 sequential or loosely synchronized（顺序或弱同步），真实协作里却常有 temporal overlap（时间重叠）和 role coupling（角色耦合）。Co-Motion 是一种 exploratory demonstration paradigm（探索性示范范式），目标是在训练集里增加 concurrent bimanual coordination 密度。

![[papers/images/2606.20285v1/co-motion.png|700]]

**Fig. 3: Sequential motion paradigm (left) vs. collaborative motion paradigm (right).** 左：传统顺序双臂运动；右：Co-Motion 把任务分 stage，在非冲突子任务上并行 dispatch 双臂，减少 idle time（空闲时间），捕获自然时间重叠。

实现上，Co-Motion 改的是 RoboTwin 2.0 代码生成管线里的 task-level scheduling logic（任务级调度逻辑），底层 cuRobo planner 和 primitive interfaces 不变。三个支撑机制：

1. **shared reference frames（共享参考系）**：如 handover midpoint、goal pose，让两臂规划解耦且跨 trial 稳定。
2. **look-ahead precomputation（前瞻预计算）**：提前算近未来交互目标，减少 stop-and-plan 中断。
3. **safe intermediate targets（安全中间目标）**：带 clearance margin 的缓冲姿态；并行不可行时 graceful fallback 到 sequential execution。

重要限定：Co-Motion 是 complementary data strategy（互补数据策略），不是 SAE/LAC 的前提。Table I 表明在 standard sequential demonstrations 上，Co-VLA 已经稳定优于 baseline。

## IV. Experiments / 实验

### IV-A Training Pipeline / 训练流程

两阶段 fine-tuning：

| 阶段 | 设置 | 目的 |
| --- | --- | --- |
| Phase 1 warm-up | 冻结 pretrained backbone，只训 SAE 新层；1000 steps；peak LR $5\times10^{-5}$ | 让 shared/residual projections 和 routing modules 合理初始化，不破坏预训练表征 |
| Phase 2 full fine-tuning | 解冻全部参数；30000 steps；peak LR $2.5\times10^{-5}\to2.5\times10^{-6}$ | 端到端适应双臂协调结构 |
| 通用设置 | batch size 32，4 GPU，FSDP | 标准大规模微调配置 |

### IV-B Simulation Evaluation on RoboTwin 2.0

![[papers/images/2606.20285v1/robotwin_task.png|700]]

**Fig. 4: Evaluation tasks.** 上：RoboTwin 2.0 仿真（Aloha-AgileX）；下：真实 AgileX Cobot Magic 任务示意。

从 RoboTwin 2.0 的 50+ 任务中，作者**故意只选需要 concurrent bimanual motion 的子集**，覆盖 symmetric lifting、asymmetric role assignment、temporally coupled handovers 等模式。每个任务：

- 采集 **1000** 条 successful demonstrations（clean scene，无随机化）
- **per-task fine-tune** 一个模型
- 评测 **100 rollouts**，分 Easy / Hard（scene randomization 强度递增）

#### 1) Co-VLA on RoboTwin Tasks

**Table I: Success rate (%) on RoboTwin 2.0 tasks over 100 rollouts.**

| Task | $\pi_{0.5}$ Easy | $\pi_{0.5}$ Hard | $\pi_0$ Easy | $\pi_0$ Hard | Co-VLA Easy | Co-VLA Hard |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Handover Block | 44% | 10% | 64% | 7% | **91%** | 12% |
| Lift Pot | 100% | 60% | 100% | 63% | 100% | 65% |
| Pick Diverse Bottles | 95% | 18% | 91% | 13% | 95% | 16% |
| Pick Dual Bottles | 100% | 27% | 100% | 16% | 100% | 18% |
| Place Bread Basket | 92% | 28% | 70% | 46% | 69% | 48% |
| Place Bread Skillet | 38% | 7% | 63% | 8% | 70% | 8% |
| Put Object Cabinet | 70% | 19% | 79% | 8% | 81% | 5% |
| Scan Object | 45% | 6% | 41% | 3% | 50% | 2% |
| **Average** | 73% | 21.9% | 76% | 21% | **82%** | 22% |

读表要点：

- Co-VLA 把平均 Easy 成功率从 $\pi_0$ 的 76% 和 $\pi_{0.5}$ 的 73% 提到 **82%**。
- 最大增益在 **Handover Block**：$\pi_0$ 64%→91%，$\pi_{0.5}$ 44%→91%，说明 tight inter-arm coordination 最受益于 shared–residual decomposition。
- $\pi_{0.5}$ 在这些 bimanual-specific tasks 上**并不稳定优于** $\pi_0$，说明 backbone capacity alone 不足以解决双臂协调，需要 structural inductive bias。

#### 2) Effect of Co-Motion Demonstrations

并非所有任务都能做 Co-Motion：并行 dispatch 要求子任务 spatially non-conflicting，且 cuRobo 在更紧 timing 下仍保持高成功率。四个可用任务是 Handover Block、Scan Object、Place Bread Skillet、Put Object Cabinet。

**效率（数据采集）— Table II: Average time (seconds) to generate 1,000 successful demonstrations**

| Task | Sequential | Co-Motion | 降幅 |
| --- | ---: | ---: | --- |
| Handover Block | 9.47 | 8.49 | ~10% |
| Scan Object | 5.65 | 4.52 | ~20% |
| Place Bread Skillet | 5.51 | 4.59 | ~17% |
| Put Object Cabinet | 8.95 | 6.45 | ~28% |

**效率（推理执行）— Table III: Average completion time (seconds) for 50 successful rollouts**

| Task | Co-Motion $\pi_0$ | Co-Motion Co-VLA | Sequential $\pi_0$ | Sequential Co-VLA |
| --- | ---: | ---: | ---: | ---: |
| Handover Block | 28.39 | 28.56 | 32.87 | **25.38** |
| Place Bread Skillet | 15.02 | 15.28 | 12.79 | **12.35** |
| Put Object Cabinet | 26.96 | 28.69 | 20.87 | 21.08 |
| Scan Object | 15.94 | 19.00 | 12.57 | 13.92 |
| **Average** | 21.58 | 22.88 | 19.78 | **18.18** |

Co-Motion 训练出的策略往往执行更 temporally compact（时间更紧凑），与摘要“完成时间最多减少 25%”一致。

**可学习性 — Table IV: Success rate (%) when trained with Co-Motion demonstrations**

| Task | $\pi_0$ Easy | $\pi_0$ Hard | Co-VLA Easy | Co-VLA Hard |
| --- | ---: | ---: | ---: | ---: |
| Handover Block | 15% | 8% | 38% | 8% |

相对 sequential demonstrations（Table I 平均 Easy 76% / 82%），Co-Motion 数据显著更难学：$\pi_0$ 50%，Co-VLA 56%。但 Co-VLA 仍优于 $\pi_0$，说明 SAE 更能吸收 high-density coordination signals。总体是 **efficiency-vs-learnability trade-off**：数据更紧凑、更 rich，但当前 VLA 架构还吃不满。

### IV-C Real-World Evaluation / 真实世界评测

平台：**AgileX Cobot Magic** 双臂机器人，wrist-mounted cameras（腕部相机）。每任务 **50** 条 human teleoperation demonstrations，带 randomized object positions and poses。

- **ID**：沿用训练时随机化协议。
- **OOD**：额外加入 cluttered backgrounds、distractor objects、low-lighting。

![[papers/images/2606.20285v1/real_task.png|700]]
![[papers/images/2606.20285v1/real_task-OOD.png|700]]

**Fig. 5: OOD conditions for Lift Pot.** 真实 OOD 包括杂乱背景、干扰物体、弱光等。

LAC 真实部署参数（全任务固定）：

$\alpha_{\mathrm{base}}=0.4$, $\Delta_{\mathrm{macro}}=0.4$, $\Delta_{\mathrm{prec}}=0.3$, $\Delta_{\mathrm{noise}}=0.1$, $\tau_\rho=0.01$, $\tau_\omega=0.3$, $\beta=0.2$, $[\alpha_{\min},\alpha_{\max}]=[0.05,0.95]$。

比较三种变体：$\pi_0$、Co-VLA without LAC（Co-VLA$_{\text{noLAC}}$）、Co-VLA with LAC（Co-VLA$_{\text{LAC}}$）。

**Table V: Real-world experiment results over 30 rollouts**

| Method | Handover ID | Handover OOD | Pick Bottles ID | Pick Bottles OOD | Lift Pot ID | Lift Pot OOD |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| $\pi_0$ | 63% | 13% | 57% | 37% | 43% | 33% |
| Co-VLA$_{\text{noLAC}}$ | 57% | 17% | 57% | **50%** | 63% | 27% |
| Co-VLA$_{\text{LAC}}$ | **73%** | **27%** | **67%** | 47% | **67%** | **37%** |

关键解读：

- 完整系统 Co-VLA$_{\text{LAC}}$ 在 **6 个设置里 5 个最高**。
- 只看 SAE：Lift Pot ID 43%→63% 明显提升，但 Handover ID 63%→57% 反而下降，说明 **structured decomposition alone 并不均匀有效**。
- 打开 LAC 后，所有 ID 设置都恢复并超过 $\pi_0$；OOD 上 Handover 13%→27%、Lift 33%→37%，与摘要“OOD 翻倍”一致。
- 例外：Pick Bottles OOD 上 noLAC 50% 高于 LAC 47%，可能因为 LAC 的 noise suppression 在 distribution shift 下误伤了有益 residual。

结论：**SAE 提供 representational structure，LAC 把它翻译成 reliable execution，两者互补。**

### IV-D In-Depth Analysis / 深入分析

#### a) Latent–Behavior Alignment

问题：structured latent decomposition 是否真的对应行为上的 coordination？

做法：计算 inter-arm synchronization $\mathrm{Sync}_t$，即最终组合动作 $a_{t,L}$ 与 $a_{t,R}$ 的余弦相似度；再观察它与 $E_t^s$、$E_t^r$ 的关系。

![[papers/images/2606.20285v1/norm_s_vs_bimanual_coordination_pick.png|700]]
![[papers/images/2606.20285v1/norm_r_vs_bimanual_coordination_pick.png|700]]

**Fig. 6: Latent–behavior alignment on Pick Bottles.** 左：$E_t^s$ 与 $\mathrm{Sync}_t$ 正相关；右：$E_t^r$ 与 $\mathrm{Sync}_t$ 负相关。说明 shared latent 像 coordination driver，residual 编码 arm-specific deviations，而不是随便切分内部表征。

#### b) Action Head Ablation and Structural Priors

**Table VI: Auxiliary loss ablation (Easy, 100 rollouts)**

| Aux. Loss | Lift Pot (Sym.) | Put Object Cabinet (Asym.) | Place Bread Skillet (Temp.) |
| --- | ---: | ---: | ---: |
| None | 99% | 49% | 51% |
| + $\mathcal{L}_{\text{sparse}}$ | 100% | 51% | 55% |
| + $\mathcal{L}_{\text{shared}}$ | 100% | **72%** | 55% |
| + $\mathcal{L}_{\text{sync}}$ | 100% | 70% | **62%** |

任务依赖性很明显：

- 对称搬运 Lift Pot：无辅助损失已接近饱和。
- 角色不对称 Cabinet：$\mathcal{L}_{\text{shared}}$ 49%→72% 增益最大。
- 时间耦合 Skillet：$\mathcal{L}_{\text{sync}}$ 51%→62% 最有效。

作者因算力限制只评估 **single-loss variants**，未探索组合与自适应权重。

#### c) Effect of LAC

在真实 Pick Bottles 上，对比 Co-VLA+LAC、Co-VLA no LAC、naive EMA baseline。EMA 轨迹最平滑，但成功率只有 **60%**，低于 LAC 的 **67%**。

![[papers/images/2606.20285v1/acc-zoom-in.png|700]]

**Fig. 7: Effect of LAC on joint trajectories.** LAC 降低 acceleration peaks（加速度尖峰）和 cross-rollout variance（跨回合方差），同时保留 task-critical motion patterns。朴素 EMA 因 uniform low-pass 带来 phase lag（相位滞后），会冲掉 precision-critical micro-adjustments。

## V. Conclusion / 结论

论文总结：Co-VLA 通过 SAE 把 coordination 分解成 shared / residual latents，再用 LAC 在部署时提升 success、smoothness 和 latent interpretability。

作者自己承认的局限与未来方向：

1. shared–residual decomposition 虽 architecture-agnostic，但在 Diffusion Policy 等其他 continuous-action 范式上的验证仍是 future work。
2. Co-Motion 虽丰富 concurrent samples，但相对 sequential demonstrations 成功率下降，说明 tighter temporal coupling 对 VLA 的 spatio-temporal reasoning 要求更高。
3. auxiliary loss ablation 仅限单损失 + 人工 per-task 选择；未来可用 inter-arm velocity correlation、role asymmetry indices 等 **computable task-level coordination descriptors（可计算任务级协调描述符）** 做 automatic loss routing，并探索多损失联合与 adaptive weighting。

## 讨论、局限与可追问点

### 方法层面

- **任务损失路由仍偏人工**：Table VI 证明不同 $\mathcal{L}_{\text{aux}}$ 有效，但选择依赖对任务协调模式的先验。若你自己的双臂任务没有明显“对称/不对称/时间耦合”标签，怎么自动选损失？
- **SAE 与 LAC 非单调互补**：真实 Handover 上 noLAC 反而低于 $\pi_0$，说明“结构化”不等于“更好”，执行层 refinement 是关键。能否把 LAC 的部分逻辑蒸馏回训练期？
- **14 维 joint velocity 假设**：方法绑定 $\pi_0$ 式 joint-space action interface。若换成 end-effector delta pose、operational space action 或 tactile-conditioned action，shared/residual 语义是否仍成立？

### 实验层面

- **per-task fine-tuning**：每个 RoboTwin 任务单独训一个模型，难直接说明 multi-task generalist 场景下的收益。
- **Hard setting 增益有限**：Table I 平均 Hard 仅 21%→22%，说明 scene randomization 下协调结构优势可能被感知/规划瓶颈掩盖。
- **Co-Motion 负收益未被完全解决**：采集更快但学习更难，论文只揭示 trade-off，没有给出“如何既高密度又易学”的算法答案。

### 工程与复现

- 无公开 code / project page / dataset link，LAC 阈值和 SAE 层细节需回查正文与附录。
- LAC 固定参数跨任务，Pick Bottles OOD 的退化提示 **deployment-time hyperparameters** 可能需要 task-adaptive 化。

### 与自己研究的连接点

- 若你做 **双臂 VLA / 双手+触觉策略**，SAE 的 additive interface 很值得借鉴：不改 I/O，只改 action head 结构。
- 若你关心 **UMI / robot-free demonstrations** 的质量，Co-Motion 更像“示范调度器”思路：不是录更多，而是录到更多 concurrent coordination modes。
- 若你把 [[@tang2026frs]] 的 steering 与本文对照：FRS 在 noise/action 层调用 generalist prior；Co-VLA 在 action representation 层显式拆协作结构，再在 joint command 层用 LAC 精炼。
