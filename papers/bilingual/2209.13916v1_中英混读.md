# DTact: A Vision-Based Tactile Sensor that Measures High-Resolution 3D Geometry Directly from Darkness

## 核心词汇速查

- vision-based tactile sensor（视觉触觉传感器）：用相机观察传感器内部材料形变，从而推断接触信息的触觉传感器。
- 3D geometry（3D 几何）：接触物体在触觉表面上的深度、形状、纹理等几何信息。
- photometric stereo（光度立体）：通过不同方向光照下的图像变化估计表面法线/几何的方法。
- semitransparent elastomer（半透明弹性体）：本文的关键材料层，受压变薄后反射光减少，图像变暗。
- absorptive layer（吸收层）：黑色硅胶层，用来吸收入射/透射光，减少外界光干扰。
- intensity variation（强度变化）：接触图像与参考图像的灰度差，记作 $I_{\Delta}(u,v)$。
- depth map（深度图）：每个像素对应的受压深度，记作 $D(x,y)$。
- mapping list（映射表）：把灰度变化直接映射为深度的查表函数 $M$。
- non-planar surface（非平面表面）：球面、柱面等非平坦触觉接触面。
- ICP / Iterative Closest Point（迭代最近点）：用点云配准估计物体位姿的方法。

## 摘要

这篇论文提出 DTact，一种 low-cost（低成本）、easy-to-fabricate（易制造）的 vision-based tactile sensor。核心想法是：不再像 GelSight 那样依赖精细的内部 RGB 光照和 photometric stereo，而是利用 semitransparent elastomer 的反射特性。接触区域被压得越深，在相机图像中越暗，因此可以把 darkness（暗度）直接映射为 3D depth（深度）。

作者声称 DTact 有几个优势：只需要 single image calibration（单图标定），能进行 high-resolution 3D geometry reconstruction（高分辨率 3D 几何重建），对 illumination conditions（光照条件）更鲁棒，并且较容易扩展到 non-planar contact surfaces（非平面接触面）。下游任务上，作者展示了 pose estimation（位姿估计）和 object recognition（物体识别）。

## 1. Introduction

触觉感知对机器人操作很重要，因为它能提供 3D geometry、force、slip、hardness、temperature 等接触信息。这里作者特别强调 3D geometry，因为它能帮助推断物体 shape（形状）、pose（位姿）、texture（纹理）和 normal force（法向力）。

已有 GelSight-style sensors 能高分辨率测量接触几何，但 photometric stereo 对内部照明方向和均匀性要求很严格。这个要求带来两个问题：

- mechanical design（机械设计）和 assembly（装配）更复杂；
- 很难扩展到 non-planar shapes，所以许多传感器只能做 flat surface（平面表面）。

DTact 的基本转向是：不估计 RGB 光照梯度，而是让材料本身把深度编码成亮度。压得越深，图像越暗。这样，深度估计可以从复杂的光度立体问题变成 intensity-to-depth mapping（强度到深度映射）问题。

## 2. Related Work

论文把相关工作分成两类。

第一类是 GelSight 及其变体。这些方法通过 photometric stereo 测量高分辨率接触几何，在 texture recognition、dexterous manipulation、shape mapping 等任务中有效。但它们需要精确布置 RGB LEDs、控制光路、处理反射层，复现门槛高，也不利于非平面扩展。

第二类是试图降低光照依赖的方案。例如 DelTact 用 optical flow（光流）估计 dense color pattern（密集彩色图案）的变形；Soft-bubble 使用 depth sensor；GelStereo 使用 binocular camera（双目相机）和学习方法。这些方案各自会在 accuracy、sensor size、cost 或 algorithmic complexity 上做取舍。

DTact 的定位是：仍然使用 monocular camera（单目相机），但只用简单外部照明和材料层设计来完成高分辨率深度测量。

## 3. Design and Fabrication

DTact 分为三个模块：

- perception module（感知模块）：sensor base、USB camera、LED ring。相机分辨率为 800 x 600，60 FPS，视场角 120 度；LED ring 使用 WS2812B-8 提供白光。
- diffusion module（扩散模块）：sensor shell、acrylic plate、transparent elastomer。透明弹性体用于扩散 LED 光，减少过度反射。
- contact module（接触模块）：semitransparent layer 和 absorptive layer。半透明层在变形时亮度变化；黑色吸收层吸收穿过半透明层的光，也阻挡外部光。

![[papers/images/2209.13916v1/design_page1.png|700]]

这张结构图要和上面的三个模块一起看：相机和 LED ring 负责成像与照明，transparent elastomer（透明弹性体）负责扩散光，真正把接触深度编码成灰度变化的是 semitransparent layer（半透明层）与 absorptive layer（吸收层）的组合。

关键材料逻辑是：当接触区域被压下，半透明层局部变薄，反射回来的光减少，穿过去的光被黑色吸收层吸收，所以相机看到该区域变暗。也就是：

> pressed deeper -> darker in the output image（压得越深，输出图像越暗）

![[papers/images/2209.13916v1/principle_page1.png|700]]

这张图对应本文最核心的 physical encoding（物理编码）：同一个接触区域随着压入深度增加，图像中的暗斑逐渐变大、变深。后面的 reconstruction（重建）不是先估计表面法线，而是直接利用这个单调关系，把 intensity variation（强度变化）映射到 depth（深度）。

制造流程也比较直接：固定 acrylic plate，倒入透明硅胶并固化；再用 3D-printed open mold 固定接触模块位置，倒入半透明硅胶；固化后涂黑色硅胶作为吸收层。论文给出的传感器尺寸约为 45mm x 45mm x 47mm，不含相机的组件和模具成本低于 7 美元。

![[papers/images/2209.13916v1/fabrication_page1.png|700]]

制造图说明 DTact 的优势主要来自结构简化：它没有复杂的多方向 RGB 光路，主体部件更像“相机 + 光源 + 多层软材料”的组合，因此 fabrication（制造）和后续更换 contact module（接触模块）都更直接。

## 4. 3D Shape Reconstruction

### 4.1 Reconstruction Methods

目标是从 pixel intensity（像素强度）恢复 pressed depth（受压深度）。记：

- $I_{\Delta}(u,v)$：像素 $(u,v)$ 处接触图像和参考图像的强度变化；
- $D(x,y)$：触觉表面对应位置 $(x,y)$ 的受压深度。

理想方法是建立一个高维 lookup table：

$$
D(x,y) = R(I_{\Delta}(u,v), u, v)
$$

但这个方法需要 CNC machine（高精度数控设备）提供每个像素的连续深度，标定复杂。作者因此提出两个简化方法。

Single image method（单图方法）把映射简化成只依赖灰度变化的 mapping list：

$$
D(x,y) = M(I_{\Delta}(u,v))
$$

Linear regression method（线性回归方法）考虑照明不均匀性，为每个像素拟合一个斜率：

$$
D(x,y) = k(u,v) \cdot I_{\Delta}(u,v)
$$

又因为参考图像中心更亮，作者把 $k(u,v)$ 建模为到图像中心距离的线性函数：

$$
k(u,v) = k_c \times \sqrt{(u-u_c)^2 + (v-v_c)^2} + b_c
$$

最终：

$$
D(x,y) = \left(k_c \times \sqrt{(u-u_c)^2 + (v-v_c)^2} + b_c\right) \times I_{\Delta}(u,v)
$$

### 4.2 Calibration

标定时，作者用一个 metal ball（金属球）压 DTact。球的几何连续且半径已知，因此可以通过圆检测算法得到真实 depth map，再和图像强度变化对应起来。

作者比较 RGB 和 grayscale（灰度）通道的参考图像噪声，发现 grayscale channel 最稳定，因此后续都用灰度做标定和重建。

![[papers/images/2209.13916v1/calibration_page1.png|700]]

标定图可以按两层来读：第一层是用已知半径的金属球得到 ground-truth depth（真实深度）；第二层是把球压痕图像里的 grayscale intensity（灰度强度）和真实深度对应起来，形成后续使用的 mapping list（映射表）。这也是 single image calibration（单图标定）为什么可行的关键。

标定设置：

| 用途 | Image amount | Ball radius | Pressed place |
| --- | ---: | ---: | --- |
| Single image calibration | 1 | 4.0 mm | Near center |
| Linear regression calibration | 30 | 4.0 mm | Randomly |
| Reconstruction test | 20 | 5.0 mm | Randomly |

### 4.3 Reconstruction Pipeline

重建流程可以理解为六步：

1. capture contact image（采集接触图像）；
2. rectify image（用相机内参和畸变参数校正）；
3. crop to sensing field（从 800 x 600 裁剪到 580 x 580，对应 24 x 24 mm^2 感知区域）；
4. convert to grayscale（转灰度）；
5. subtract reference image（减去参考图像，得到 $I_{\Delta}$）；
6. map to depth and smooth（映射深度并用 7 x 7 Gaussian filters 平滑两次）。

![[papers/images/2209.13916v1/reconstruction_pipline_page1.png|700]]

这张 pipeline 图对应上面的六步：从 contact image（接触图像）开始，经过 rectification / cropping / grayscale conversion（校正、裁剪、灰度化），再通过 reference subtraction（参考图像差分）得到 $I_{\Delta}$，最后查表生成 depth map（深度图）并转成 point cloud（点云）。

实现上使用 OpenCV 和 Open3D。作者报告在 Intel Core i7-8550U @ 1.80GHz 上，不使用 GPU，rectifying、mapping、smoothing 分别约 8 ms、3 ms、14 ms，即从原始图像得到最终 depth map 约 25 ms；带可视化时约 20 Hz。

### 4.4 Reconstruction Results

标准设置下，作者用不同半径的金属球测试重建质量，指标是 MAE（mean absolute error，平均绝对误差）：

| Method | MAE (mm) |
| --- | ---: |
| Single image | 0.0476 |
| Linear regression | 0.0534 |

结果显示 single image method 更好。作者解释为：DTact 的 diffusion module 已经让参考图像足够均匀，因此全局 mapping list 可以适用于所有像素；linear regression 的拟合过程反而引入额外误差。

定性结果中，DTact 对 ball array、3D-printed star、USB cable head、screw cap、M5 hexagonal screw 的重建形状与视觉外观对齐较好。

![[papers/images/2209.13916v1/reconstruction_show_page1.png|700]]

这里的定性结果是理解 DTact 输出能力的重点：左侧是接触造成的 tactile image（触觉图像）/ intensity pattern（强度模式），右侧是对应恢复出的 depth / 3D geometry。阅读时可以重点看边缘、孔洞、螺纹这类细节是否被保留下来。

## 5. Experiments

### 5.1 Robustness to Illumination Conditions

作者改变 LED ring 的照明配置，测试 DTact 是否对 illumination uniformity（光照均匀性）鲁棒。结果如下：

| Setting | Std of reference image | Single image MAE | Linear regression MAE |
| --- | ---: | ---: | ---: |
| Standard | 4.080 | 0.0476 | 0.0534 |
| Scheme 1 | 5.237 | 0.0509 | 0.0542 |
| Scheme 2 | 5.262 | 0.0524 | 0.0586 |
| Scheme 3 | 5.451 | 0.0650 | 0.0678 |
| Scheme 4 | 16.550 | 0.0510 | 0.0651 |

Single image method 在所有 LED 配置下 MAE 都更低，说明 DTact 的深度重建对 illumination uniformity 的变化比较稳健。

![[papers/images/2209.13916v1/illumination_page1.png|700]]
![[papers/images/2209.13916v1/illumination_page2.png|700]]

这组图对应 illumination robustness（光照鲁棒性）实验。读表格时只看 MAE 会觉得差异不大；配合图看，重点是不同 LED pattern（照明模式）下 reference image（参考图像）的均匀程度变化明显，但 single image method 仍能恢复可用的 depth map。

作者还测试了 side lighting（侧向照明），即 LED 从左右侧照射，而不是常见的底部照明。定性结果显示，使用相同 pipeline 仍能得到 depth map 和 3D reconstruction。

### 5.2 Layer Thickness

作者制作了不同 semitransparent layer thickness（半透明层厚度）的传感器：1.0 mm、1.5 mm、2.0 mm、2.5 mm、3.0 mm，并用 M3、M5、M8 set screws 压测。

结论是 thinner layers（更薄的层）能捕捉 finer geometry（更细几何），但可测深度范围受层厚限制。因此实际应用需要在 geometry fineness（几何细节）和 depth range（深度范围）之间权衡。

![[papers/images/2209.13916v1/texture_page1.png|700]]

这一图可以作为 layer thickness trade-off（层厚权衡）的直观补充：DTact 能看到纹理/螺纹类细节，但细节分辨率和可压入范围由半透明层厚度共同决定。也就是说，材料参数本身就是 sensing range（感知范围）和 sensitivity（灵敏度）的设计变量。

### 5.3 Non-planar Surface

作者把 DTact 的平面接触面扩展成 sphere（球面）和 cylinder（柱面）两类 non-planar surfaces。由于 diffusion module 仍能保持较均匀的参考图像，原来的 depth map 计算方法可以复用。

最后通过 ray casting algorithm（射线投射算法）把 depth map 投影到非平面表面，实现非平面重建。这个实验的意义是：DTact 的机械和算法设计比 GelSight-style sensors 更容易适配特殊末端执行器形状。

![[papers/images/2209.13916v1/non-planar_page1.png|700]]

非平面图要看两点：一是 sphere / cylinder（球面/柱面）接触面仍然能得到较均匀参考图像；二是 depth map 不再直接解释为平面高度，而要通过 ray casting 投影回曲面坐标。这里展示的是可扩展性证明，不是大规模定量评估。

## 6. Applications

### 6.1 Pose Estimation

作者用 DTact 生成的 3D point cloud 进行 pose estimation。方法是把重建点云输入 ICP（Iterative Closest Point），跟踪一个 long M3 nut 从左侧绕到后方再到右侧的旋转过程。

定性结果显示 predicted poses（预测位姿）和 actual poses（一致）较好。实际运行中，Open3D 的 ICP 在同一台桌面 CPU 上约 10 Hz。

![[papers/images/2209.13916v1/pose_estimation_page1.png|700]]

位姿估计图说明 DTact 的 depth output（深度输出）可以直接进入下游几何算法。这里不是训练一个端到端 pose network（位姿网络），而是把重建点云交给 ICP 做配准；因此这个应用更像是在证明 3D reconstruction 本身足够稳定。

### 6.2 Object Recognition

作者收集了 4,320 张 tactile images，覆盖 12 类物体，包括 metal balls、cylinders、nuts、bolts、spanners、legos、USB cable heads、keys、hex keys、mesh pen holders 等。每个物体通过不同压力生成不同深度的触觉图像。

模型使用 ResNet-18，训练/测试划分为 70% / 30%，训练集上做 4-fold cross validation（四折交叉验证）。训练使用 cross-entropy loss、Adam optimizer，学习率从 1e-3 开始，每 5 epoch 乘以 0.3，并使用 random crop、random flip、random rotate 做数据增强。

结果：15 epochs 后测试集 accuracy（准确率）约 96%。这说明 DTact 捕捉到的 tactile image / depth features 对物体类别有区分度。

## 7. Conclusion

DTact 的核心贡献不是提出复杂算法，而是用一个材料和结构设计降低了 vision-based tactile sensing 对 illumination 的依赖。它把“接触深度”转成“图像变暗程度”，用单张图完成标定，再把灰度变化直接映射到 depth map。

这篇论文值得注意的点：

- 如果你关心 tactile sensor 的可制造性，DTact 的结构很有参考价值。
- 如果你关心机器人下游任务，pose estimation 和 object recognition 展示了它能输出可用的 3D/tactile features。
- 如果你关心系统设计，这篇文章的 trade-off 很清楚：材料层越薄，细节越好；但可测深度范围会变小。
