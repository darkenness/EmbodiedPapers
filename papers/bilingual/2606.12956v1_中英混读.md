## 论文主线

这篇论文解决的是 VLA 在长时程移动操作中的“状态记忆”问题。桌面短任务里，当前图像通常已经包含任务所需的大部分线索；但移动操作中，机器人会离开目标、遮挡对象、移动物体，还要在不同房间区域之间切换。策略必须持续回答三个问题：**Where am I?（我在哪里）**、**What has changed?（环境发生了什么变化）**、**How far along am I?（任务进度到哪一步）**。SERF 的回答是：不要把这些都压进短历史图像序列，而要维护一个能随对象和机器人运动更新的 spatiotemporal feature map（时空特征图）。

![[papers/images/2606.12956v1/teaser.jpg|700]]

**Fig. 1: SERF teaser.** 上半部分展示移动机械臂执行多子目标长时程任务；下半部分用 PCA 可视化同一 latent space 中的 environment points（环境点）和 robot points（机器人点）。这张图的核心是：map 不是静态 3D reconstruction，而是随着 egocentric observations（第一视角观察）和 proprioceptive state（本体状态）在线更新。

SERF 的方法分三层。第一层是 map representation：把 RGB-D 图像中的 DINOv3 patch embeddings back-project（反投影）到 3D，注册为 environment neural points；同时从机器人 URDF/mesh 采样 robot neural points。两类点共享 decoder，因此特征落在同一个 latent space。第二层是 map update：环境点按 object-level $\mathrm{SE}(3)$ tracking 移动，机器人点按 forward kinematics 移动。第三层是 policy interface：map tokenizer 从多参考系、多尺度抽取 8 个 map tokens，接到 $\pi_{0.5}$ VLA 的 action expert，让策略同时看到当前图像和结构化空间记忆。

实验验证围绕 BEHAVIOR-1K 三个家庭任务：Collecting Children's Toys、Putting Shoes On Rack、Assembling Gift Baskets。结果显示，SERF 相比 image-only $\pi_{0.5}$ fine-tuned baseline，把三任务平均 task progress 从 44.0% 提到 58.7%；在 OOD scene shifts（目标位置移动、目标数量增加、目标放到未访问区域）和 object-drop failure recovery（物体掉落恢复）中也更稳。

## 摘要与核心贡献

摘要的逻辑是：long-horizon robot mobile manipulation（长时程机器人移动操作）需要持续推理 localization（定位）、environment changes（环境变化）和 task progress（任务进度），这些很难只从 image observations（图像观察）中推断。SERF 通过 spatiotemporal feature map 把环境和机器人本体表示为 shared latent space（共享潜在空间）中的 neural points，并从 egocentric observations 和 proprioception 在线更新。

核心贡献可以压缩成三点：

- **表征贡献**：环境和机器人本体不再分开处理，而是共同进入一个 neural point map；这让策略能直接看到 robot-object spatial relationships（机器人-对象空间关系）。
- **更新机制**：environment points 用 object-level $\mathrm{SE}(3)$ tracking 更新，robot points 用 forward kinematics 更新；训练好的 latent features 固定，执行时主要移动点坐标。
- **VLA 接口**：map tokenizer 把 map 压成多参考系、多尺度 tokens，和 image/state/task tokens 拼接，训练一个 map-conditioned $\pi_{0.5}$ action expert。

实验结论对应这三点：显式 map memory 提升 task progress；temporal map update 比 static feature map 更好；加入 robot points 又比 environment-only map 更好；在场景配置变化和掉落恢复中，SERF 更能利用“过去知道的对象在哪里”。

## 1. Introduction / 为什么长时程移动操作需要 SERF

Introduction 用一个整理儿童房的例子切入：移动机械臂要收集散落玩具、绕过家具、把它们放到书架上。这个任务不只是“看到物体就抓”，而是多阶段、多对象、跨区域、会改变场景状态的过程。机器人要知道自己移动到了哪里、哪些对象已被移动、哪些子目标已经完成。

作者把现有方法分成几类：

- **Modular mobile manipulation systems（模块化移动操作系统）**：有显式 perception/navigation/planning/manipulation 模块，也可能有 3D scene graph 或 object memory。优点是可解释；缺点是对象/符号抽象可能丢掉 dense geometry（稠密几何）和连续控制所需的交互线索。
- **Policy-internal memories（策略内部记忆）**：episodic memories、history tokens、retrieval buffers、language-indexed memory 等。优点是能延长时间上下文；缺点是 spatial grounding（空间落地）常是隐式的，策略要自己从历史中猜哪些观察和当前状态相关。
- **3D feature maps（3D 特征地图）**：把 VFM features 放到空间中，提供对象和目标的空间记忆。但已有 feature-map 方法常只表示 static environment（静态环境），不跟踪机器人交互后对象怎么移动，也不显式表示 robot body。

SERF 的定位是：做一个随时间演化的 4D feature map，同时包含 environment 和 robot body。它支持两种推理：

- **Allocentric reasoning（以世界为中心的推理）**：map 保存全局位置、目标位置、对象移动轨迹。
- **Egocentric reasoning（以机器人为中心的推理）**：robot points 暴露 gripper、base、对象之间的距离、方向和可达关系。

Introduction 还给出关键数字：在三个 BEHAVIOR-1K 任务上，SERF 把 image-only fine-tuned baseline 的平均 task progress 从 $44.0\%$ 提到 $58.7\%$；在 unvisited region OOD 设置中从 $28.0\%$ 提到 $51.0\%$；还能从 object-drop failures 中恢复。

## 2. Problem Formulation / 问题定义

本文考虑一个 mobile manipulator 在 workspace $\mathcal{X}\subseteq \mathbb{R}^3$ 中操作。每个时间步 $\tau$，机器人有：

- proprioceptive state（本体状态）$s_\tau$，包括 base pose 和 joint angles；
- head/wrist cameras 的 RGB-D observations：$o_\tau=\{(I_\tau,Z_\tau)\}$；
- instance segmentation labels（实例分割标签）$C_\tau$；
- VFM patch embeddings：$Y_\tau\in\mathcal{Y}^{H\times W}$，其中 $\mathcal{Y}\subseteq\mathbb{R}^k$。

RGB-D 和 camera pose 可以把 patch centroid 反投影成 3D coordinate，从而得到 coordinate-embedding pairs $(x,y)\in\mathcal{X}\times\mathcal{Y}$。任务由 natural language command $\ell$ 指定，实际进入策略的是 task embedding $e_\ell$。

作者把学习问题拆成两个：

### Problem 1: Spatiotemporal Feature Mapping

学习 map parameters $\Theta$，让 map $m_\tau$ 能在 workspace 坐标 $x$ 处重建 VFM embedding $y$：

$$
\mathcal{L}_{map}(\Theta)=
\mathbb{E}_{\tau,\,(x,y)\sim\mathcal{D}_\tau}
\left[\mathcal{L}(m_\tau(x;\Theta),y)\right].
$$

直观含义：给定一个空间点 $x$，SERF map 应该知道那里对应什么视觉语义/外观特征。$\mathcal{D}_\tau$ 是到时间 $\tau$ 为止由 onboard observations 产生的 coordinate-embedding samples；$\mathcal{D}_0$ 是执行前的 prior reconstruction dataset。

### Problem 2: Map-Conditioned Behavior Cloning

在专家示范 $\mathcal{T}$ 上学习策略：

$$
\mathcal{L}_{bc}(\phi)=
-\mathbb{E}_{(o_\tau,s_\tau,a_\tau,\ell)\sim\mathcal{T}}
\left[\log \pi_\phi(a_\tau \mid m_\tau,o_\tau,s_\tau,e_\ell)\right].
$$

和标准 BC 的区别是：策略不只看当前 observation/state/task，还显式看 map $m_\tau$。这就是本文的核心接口，map 是 history 的压缩形式，而不是把完整历史帧直接喂给策略。

## 3. Spatiotemporal Feature Mapping / SERF 地图怎样构建

![[papers/images/2606.12956v1/overview_mapping_page1.png|700]]

**Fig. 2 right: Neural point features are interpolated and decoded to reconstruct per-patch VFM embeddings.** 这张图展示 neural point map 的查询过程：附近点的 latent features 被插值，然后 decoder 重建 VFM embedding。

### 3.1 Neural Point Representation / 神经点表示

SERF 把 map 写成 $m_\tau:\mathcal{X}\to\mathcal{Y}$，即从 workspace coordinate 到 VFM embedding space 的函数。它不是显式 voxel grid，而是一组 neural points：

$$
\mathcal{P}_\tau = \{(p_{i,\tau}, f_i, c_i)\}_{i=1}^{N}.
$$

其中：

- $p_{i,\tau}\in\mathcal{X}$ 是第 $i$ 个点在时间 $\tau$ 的 world-frame position；
- $f_i\in\mathcal{F}$ 是 learnable latent feature；
- $c_i$ 是 object instance label；
- environment points 和 robot points 合起来构成 $\mathcal{P}_\tau=\mathcal{P}^e_\tau\cup\mathcal{P}^r_\tau$。

查询位置 $x$ 时，SERF 从 ball query 中取 $K$ 个近邻，并用距离 softmax 插值：

$$
F(x;\mathcal{P}_\tau) =
\sum_{i\in\mathcal{N}(x;\mathcal{P}_\tau)}
w_i(x;\mathcal{P}_\tau)\, f_i,
$$

$$
w_i(x;\mathcal{P}_\tau)=
\mathrm{softmax}\left(-\|x-p_{i,\tau}\|/\sigma\right),\quad \sigma>0.
$$

然后用 decoder $D_\theta$ 得到 map prediction：

$$
m_\tau(x;\Theta)=D_\theta(F(x;\mathcal{P}_\tau)).
$$

直观理解：neural point 的坐标告诉 map “这个 feature 在哪里”，latent feature 告诉 map “这里是什么样的语义/外观”。因为坐标是显式的，物体移动时只要移动点坐标，feature identity 可以保留。

### 3.2 Environment Points / 环境点

环境点来自 robot observations。流程是：

1. 从 RGB image 提取 DINOv3 patch embeddings。
2. 用 depth 和 camera pose 把 patch center 反投影到 3D。
3. 对 3D points voxelize，在 occupied voxel 中注册一个 environment neural point。
4. 给每个点赋 instance label，并随机初始化 latent feature。
5. 执行前用 reconstruction loss 学 $\mathcal{P}^e_0$ 的 latent features。

环境点的 reconstruction loss 是 cosine-similarity objective：

$$
\mathcal{L}_{rec}=1-\mathrm{sim}\left(D_\theta(F(x;\mathcal{P}^e_0)),y\right).
$$

这里 $y$ 是 DINOv3 patch embedding。直观含义是：decoder 输出应尽量和视觉基础模型在该 patch 上的 embedding 方向一致。

### 3.3 Robot Points / 机器人点

机器人本体也被表示为 neural points。给定 robot kinematic model，例如 URDF，作者从 robot link meshes 的 surface 采样点：

$$
\mathcal{P}^r=\{(u_j,f^r_j,c^r,l_j)\}_{j=1}^{N_r}.
$$

其中 $u_j$ 是 link local frame 中的 surface point，$l_j$ 是该点所属 link，$c^r$ 是共享 robot-body label。给定 robot state $s$，用 forward kinematics 得到 link pose：

$$
p^r_j(s)=R_{l_j}(s)u_j+t_{l_j}(s),
$$

其中 $(R_{l_j}(s),t_{l_j}(s))\in\mathrm{SE}(3)$。因此 robot point 的 world-frame 位置不是学出来的，而是由机器人本体状态直接决定。

这个设计的意义是：策略不需要仅从 proprioception 数字中隐式推断机器人身体在哪里，而能在 map 中看到 robot body 和 objects 的空间关系。

### 3.4 Latent Feature Learning / latent feature 怎样学

环境点和机器人点共享 decoder $D_\theta$，这迫使它们进入同一个 latent feature space。总损失是：

$$
\mathcal{L}_{map}=
\mathcal{L}_{rec}
+\lambda_{inter}\mathcal{L}_{inter}
+\lambda_{intra}\mathcal{L}_{intra}.
$$

这里：

- $\mathcal{L}_{rec}$ 让 map reconstruction 对齐 DINOv3 embeddings；
- $\mathcal{L}_{inter}$ 让 same-category features 聚在一起，不同 category 分开；
- $\mathcal{L}_{intra}$ 让 same-part features 聚在一起，同一 instance 的不同 part 分开。

![[papers/images/2606.12956v1/inter_contrastive.jpg|700]]

**Fig. Inter-category contrastive visualization.** PCA projection 显示同类 features 在不同场景中聚合，不同类 features 分离。这里 robot features 被当作额外 category。

![[papers/images/2606.12956v1/intra_contrastive_page1.png|700]]

**Fig. Intra-instance contrastive visualization.** PCA projection 显示同一 instance 内 same-part features 聚合，不同 part 分开。这个结构对抓取、放置等需要部位级线索的操作有价值。

### 3.5 Map Updates / 地图怎样随时间更新

执行开始时，$m_0$ 由 environment points $\mathcal{P}^e_0$、robot points $\mathcal{P}^r(s_0)$ 和 decoder $D_\theta$ 定义。之后只更新点坐标，latent features、labels 和 decoder 固定。

环境更新假设每个 movable object instance 是 rigid body。对某个 instance 的点集 $\mathcal{I}$，先在连续 observations $(\tau-1,\tau)$ 中 track 2D keypoints，再 lift 到 3D，估计刚体运动：

$$
(\hat{R},\hat{t})=
\arg\min_{R\in\mathrm{SO}(3),\,t\in\mathbb{R}^3}
\sum_k \|q^\tau_k-(R q^{\tau-1}_k+t)\|^2.
$$

实现上用 Fast Global Registration (FGR) 初始化，再用 Iterative Closest Point (ICP) refine。然后对该 instance 的所有环境点更新：

$$
p^e_{i,\tau}=\hat{R}p^e_{i,\tau-1}+\hat{t},\quad i\in\mathcal{I}.
$$

机器人更新则直接用当前 $s_\tau$ 通过 forward kinematics 计算每个 link transform。最终：

$$
\mathcal{P}_\tau=\mathcal{P}^e_\tau\cup\mathcal{P}^r_\tau.
$$

这就是 SERF 中 “spatiotemporal” 的来源：features 表示语义/外观 identity，coordinates 表示它们在时间中的位置变化。

## 4. Map-Conditioned VLA Policy / 地图怎样接入 VLA

![[papers/images/2606.12956v1/framework_page1.png|700]]

**Fig. Overview of map-conditioned VLA policy.** A map tokenizer produces map tokens across multiple reference frames and spatial scales. The VLA model is conditioned on these map tokens, along with image observations, the task embedding, and proprioceptive state, to predict actions.

### 4.1 Map Tokenizer / 多参考系、多尺度 map tokens

Map tokenizer 先过滤 neural point set，只保留 robot points 和 BDDL task specification 中引用的 task-relevant object points，丢掉 background/structural points。然后 Point Transformer 编码点集，再由 8 个 parallel heads 产生 map tokens。

8 个 token 可分成 5 组：

| Token group | 数量 | 作用 |
| --- | ---: | --- |
| robot-base tokens | 3 | 以 robot base 为中心，半径 1 m、2 m、4 m，提供多尺度局部/中程上下文。 |
| end-effector tokens | 2 | 以左右 gripper 为中心，0.5 m radius ball query，支持 grasp reasoning。 |
| robot-only token | 1 | 汇总 robot body configuration。 |
| environment-only token | 1 | 汇总 current environment state。 |
| global token | 1 | 聚合所有点，提供 scene-level reasoning。 |

作者特意不把 absolute coordinates 直接放进 token features，而只用 Point Transformer 的 relative positional encodings，以减少 overfitting。

### 4.2 Policy Learning / 接入 $\pi_{0.5}$

SERF 基于 $\pi_{0.5}$。VLA 的视觉编码器处理 RGB observations，state tokenizer 处理 proprioception，task embedding $e_\ell$ 表示任务。Map tokens 被投影到 VLA token space：

$$
e_m=[\tilde{z}_1,\ldots,\tilde{z}_8].
$$

然后拼接：

$$
h_\tau=\operatorname{Concat}(E(o_\tau),e_m,e_s,e_\ell),
$$

其中 $E$ 是 vision encoder，$e_s$ 是 state tokens。Action expert 用 conditional flow-matching objective 训练。给定 expert action chunk：

$$
A_\tau=a_{\tau:\tau+H},
$$

以及 Gaussian noise $\epsilon$，定义：

$$
A^\alpha_\tau=\alpha A_\tau+(1-\alpha)\epsilon,\quad \alpha\in[0,1].
$$

训练 action expert $v_\psi$ 预测 velocity：

$$
u_\tau=A_\tau-\epsilon,
$$

损失为：

$$
\mathcal{L}_{action}=
\mathbb{E}_{(h_\tau,A_\tau)\sim\mathcal{T},\,\alpha,\epsilon}
\left[
\|v_\psi(A^\alpha_\tau,h_\tau,\alpha)-u_\tau\|_2^2
\right].
$$

推理时，从 $\hat{A}^0_\tau=\epsilon$ 开始，沿 velocity field 从 $\alpha=0$ 积分到 $\alpha=1$，得到 action chunk $\hat{a}_{\tau:\tau+H}$。

训练时冻结 VLM backbone 和 vision encoder，只在 action expert 插入 LoRA；可训练参数包括 map tokenizer、projection layer 和 LoRA parameters。这降低了 fine-tuning 成本，也尽量保留 $\pi_{0.5}$ 的已有能力。

## 5. Long-Horizon Mobile Manipulation / 实验

### 5.1 Benchmark / BEHAVIOR-1K 设置

实验在 BEHAVIOR-1K / OmniGibson 上做。每个任务使用 200 条 expert demonstrations 训练，在 20 个 evaluation configurations 上测试。指标是 task progress，即 BDDL task specification 中 completed subgoals 的比例。

| Task | 名称 | Target objects | Workspace | Max steps |
| --- | --- | --- | --- | ---: |
| Task 21 | Collecting Children's Toys | 2 dice, 2 teddy bears, 2 board games, 1 toy train | $4.4\times3.0$ m | 38,372 |
| Task 22 | Putting Shoes On Rack | 2 gym shoes, 2 sandals | $5.5\times9.0$ m | 15,384 |
| Task 26 | Assembling Gift Baskets | 4 baskets, 4 candles, 4 butter cookies, 4 Swiss cheeses, 4 bows | $7.6\times8.8$ m | 52,120 |

BDDL goal conditions：

- Task 21：所有目标玩具要放到同一个 bookcase。
- Task 22：每个 footwear item 要放在 rack 上，不能接触地面，同类别物体要相邻。
- Task 26：每个 basket 必须包含一支 candle、一块 butter cookie、一块 Swiss cheese 和一个 bow。

### 5.2 Baselines / 对比方法

作者比较 5 个策略变体：

| Method | 含义 |
| --- | --- |
| PI0.5 (pre) | 来自 Larchenko et al. 的 BEHAVIOR-1K $\pi_{0.5}$ checkpoint，已在 50 个 BEHAVIOR-1K tasks 上 fine-tune。 |
| PI0.5 (ft) | 每个目标任务单独 fine-tune 的 image-only VLA。 |
| SBP | Static 3D feature map baseline。 |
| SERF (env) | 使用 SERF map，但只含 environment points，不含 robot neural points。 |
| SERF | 完整方法，含 environment points 和 robot points。 |

所有 variants 共享同一个 $\pi_{0.5}$ backbone 和训练设置，差别主要是 map-token inputs。SBP 和 SERF (env) 只有 environment points，所以使用 6 个 tokens；robot-only token 不存在，global token 会和 environment-only token 重复。

1. map-conditioned policies 比 image-only policies 更好，说明 explicit spatial memory 有用。

3. full SERF 平均优于 SERF (env)，说明 robot neural points 提供了额外的 egocentric robot-environment spatial reasoning。

## Appendix / 附录中的重要细节

### A. Map Dataset Generation / 地图训练数据怎样生成

环境样本来自 RGB-D 和 camera pose。给定 patch center $\rho$，depth $Z[\rho]$，camera-to-world pose $(R_{cam},t_{cam})$，反投影公式是：

$$
x(\rho)=R_{cam}\,\Pi^{-1}
\begin{bmatrix}\rho\\1\end{bmatrix}
Z[\rho]+t_{cam}.
$$

这里 $\Pi^{-1}$ 用 camera intrinsics 把像素点反投影到 camera frame，再由 camera pose 转到 world frame。每个 patch 得到 $(x(\rho),y(\rho))$，其中 $y(\rho)=Y[\rho]$。

机器人样本则来自多个 representative robot states：渲染机器人、提取 patch embeddings、反投影 robot-body patches，聚合成 robot coordinate-embedding pairs。

![[papers/images/2606.12956v1/robot_mapping_page1.png|700]]

**Fig. Robot renderings from multiple viewpoints provide state-conditioned reconstruction samples.** 这张图说明 robot points 的监督并不是来自真实执行中的单一视角，而是用多个视角和多个状态覆盖机器人表面。

### B. Map Representation Details / 关键超参数

| Component | 设置 |
| --- | --- |
| VFM | DINOv3 |
| RGB input | $480\times480$ |
| DINOv3 output | $1280$-dim embeddings on $30\times30$ grid, patch size 16 |
| Part masks | SAM 2 base-plus |
| Environment voxel size | $0.02$ m |
| Robot surface sampling resolution | $0.02$ m |
| Neural point feature dim | 64 |
| Decoder | residual MLP, 64-d latent -> 1280-d DINOv3 embedding |
| Query neighbors | $K=6$ |
| Softmax temperature | 0.05 |

### C. Contrastive Objectives / 对比学习目标

Inter-category objective：

$$
\mathcal{L}_{inter}=
-\log
\frac{\exp(\mathrm{sim}(f_i,f_j^+)/\sigma_c)}
{\sum_k \exp(\mathrm{sim}(f_i,f_k)/\sigma_c)}.
$$

$f_j^+$ 是和 $f_i$ 同 category 的 positive，分母包括所有 category 的 positive/negative。作者用 category-indexed feature bank，每次最多采样 16,384 points，并进行 category-balanced importance sampling。Robot features 被当作额外 category。设置为 $\lambda_{inter}=0.02$，$\sigma_c=0.1$。

Intra-instance objective：

$$
\mathcal{L}_{intra}=
-\log
\frac{\exp(\mathrm{sim}(f_i,f_j^+)/\sigma_c)}
{\sum_{k\in parts}\exp(\mathrm{sim}(f_i,f_k)/\sigma_c)}.
$$

这里 positive 是同一 part 的 feature，分母包含同一 instance 内各 part labels 的 sampled features。环境 part labels 来自 SAM 2 segments，机器人 part labels 来自 robot links。设置为 $\lambda_{intra}=0.01$，$\sigma_c=0.1$。

### D. Map Updates and Tracking / 实现细节

训练和在线执行使用同一 map update rule。差别是：训练中 object-level $\mathrm{SE}(3)$ trajectories 可以预计算，在线执行中要从 robot observations 估计。

Object tracking 的实现要点：

- 只跟踪 movable object instances，不对 stationary objects（例如 tables）估计 $\mathrm{SE}(3)$。
- 每个 instance mask 内用 Shi-Tomasi corners 初始化 keypoints。
- 用 CoTracker3 在 head/wrist camera views 中跟踪 2D keypoints。
- 将 tracks 用 depth 和 camera pose lift 到 3D。
- 如果 instance centroid 的 3D displacement 小于 $0.015$ m，认为该 instance stationary，跳过该步 registration。

### E. VLA Policy Details / 策略训练细节

| Component | 设置 |
| --- | --- |
| Filtered point sample | 25,000 neural points per policy input |
| Shared Point Transformer | 2 stages, widths (128, 256), 2 blocks/stage, stride 4, 16-NN local attention |
| Branch heads | width 256, stride 1, 16-NN |
| Map token dimension | 2048 |
| Base VLA | BEHAVIOR-1K OpenPI implementation of $\pi_{0.5}$ |
| Action horizon | 30 |

## 局限与可追问点

### 1. 依赖 prior map 和 privileged instance labels

作者明确说，SERF 目前依赖 execution 前已经学好的 prior map，并在仿真中使用 privileged instance labels 做 map construction 和 updates。因此和 image-only VLA 的比较应该理解为：在这些假设下，加入 persistent spatial memory 带来多少收益；不是完全相同输入信号的公平控制变量实验。

可追问：真实系统中，prior map 如何快速建立？如果用 SAM 2 或其他 segmentation foundation model 替代 privileged labels，tracking 错误会怎样累积？

### 2. 真实世界移动操作尚未验证

实验都在 BEHAVIOR-1K/OmniGibson 中。虽然任务很长、很复杂，但真实移动操作会有深度噪声、相机标定误差、遮挡、对象非刚体变形、抓取失败等额外问题。

可追问：SERF 的 object-level $\mathrm{SE}(3)$ tracking 在真实家庭环境里是否足够稳定？map drift 是否会直接误导 VLA？

### 3. 动态更新假设 object-level rigid motion

SERF 把每个 movable object instance 当刚体，用一个 $\mathrm{SE}(3)$ 更新该 instance 的所有 environment points。这对鞋、玩具、篮子等近似刚体有效，但对 articulated objects（铰接物体）和 deformable objects（可变形物体）不足。

可追问：如果做衣物、线缆、袋子、软体食品等任务，SERF 应该扩展为 part-level transforms、deformable scene flow，还是改用连续时空场？

### 4. Map tokenizer 是人工设计的

当前 token groups 是人为设定：base 半径 1/2/4 m、gripper 半径 0.5 m、robot-only、environment-only、global。这个设计可解释，但可能不是不同任务的最优信息瓶颈。

可追问：能否根据 language task 或当前 subgoal 动态选择 map tokens？能否把 task-level semantics、robot state、object state 显式 disentangle？

### 5. Temporal reasoning 仍偏当前 map

SERF 维护时空 map，但 policy input 主要是当前 map tokens。作者也提到，多 temporal windows，包括 past observations 和 predicted future states，可能进一步增强 temporal reasoning。

可追问：是否需要把 map history 编码成 trajectory tokens？是否能和 world model / action model 结合，让 map 不只记录过去，也预测未来？

## 为什么需要回看这篇

这篇是 VLA 状态表征线的好入口。它的关键不是某个单独模块，而是一个清楚的接口范式：**把长时程空间记忆、对象变化和机器人本体关系压成 VLA 可读 tokens**。以后读以下方向时可以回看 SERF：

- VLA + memory：比较 history tokens、retrieval memory、scene graph memory 和 SERF map tokens。
- VLA + world model：SERF 是状态记忆，world model 可在它上面做预测或规划。
- tactile / VLA fusion：触觉信息也可以进入一个随时间更新的 robot-object map，而不一定直接输出 action。
- mobile manipulation benchmarks：BEHAVIOR-1K 的 BDDL task progress 和长 rollout 设置可作为评估参考。
- robot embodiment representation：SERF 的 robot neural points 是把机器人身体显式放进策略输入的简洁做法。
