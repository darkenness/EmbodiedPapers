## 论文主线

这篇报告的核心问题是：现有 general Vision-Language Models（通用视觉-语言模型）已经能回答图像问题、做 OCR、处理网页或文档，但真实机器人需要的能力更细、更物理、更连续。机器人要看懂 3D geometry（几何）、depth（深度）、relative spatial relations（相对空间关系）、object affordance（可供性）、trajectory（轨迹）、future state（未来状态）和 action planning（动作规划）。HY-Embodied-0.5 的目标不是只做一个更大的 VLM，而是把模型、数据和后训练都朝 embodied intelligence（具身智能）方向改造。

![[papers/images/2604.07430v1/1-teaser_page1.png|700]]

**Fig. 1: Performance of HY-Embodied-0.5 MoT-2B.** 这张 teaser 把论文主张压在一起：左侧是 embodied/spatial benchmarks，右侧是 HY-Embodied-0.5 MoT-2B VLA 的真实机器人任务成功率。重点是作者同时证明两件事：模型在视觉/空间/具身 QA benchmark 上有优势，并且这些表示能迁移到真实 robot control。

方法主线有四层。第一层是 architecture（架构）：用 HY-ViT 2.0 做原生分辨率视觉编码，用 Mixture-of-Transformers（MoT）把视觉和语言的计算路径部分解耦，再用 visual latent tokens 连接视觉全局语义和语言推理。第二层是 data（数据）：预训练阶段大规模加入 detection、depth、segmentation、pointing/counting，以及 embodied 和 spatial QA。第三层是 post-training（后训练）：先用 cold-start CoT SFT 让模型学长链推理，再用结构化 rewards 做 embodied RL，随后用 RFT 固化高质量推理，再通过 on-policy distillation 把大模型能力转给小模型。第四层是 VLA transfer（VLA 迁移）：用 MoT-2B 作为 VLA backbone，扩展 Action Expert，在真实双臂平台上完成 packing、stacking、hanging 三类操作。

这篇文献和 [[@kim2026serf]]、[[@tang2026frs]] 的关系是互补的。SERF 研究 VLA 输入侧的 explicit spatial memory（显式空间记忆）；FRS 研究怎样调用已有 VLA action prior（动作先验）；HY-Embodied-0.5 研究 VLA 前面的 embodied VLM foundation（具身 VLM 基座）如何训练。它提供的是“模型脑”的训练范式，而不是单一控制算法。

## Abstract / 摘要讲解

摘要的逻辑很直接：HY-Embodied-0.5 是面向 real-world embodied agents（现实世界具身智能体）的 foundation model family。作者认为普通 VLM 和 physical agents 的需求之间有 gap，主要体现在 spatial and temporal visual perception（时空视觉感知）以及 prediction、interaction、planning（预测、交互、规划）这类 embodied reasoning（具身推理）。

模型族有两个主要版本：

- **HY-Embodied-0.5 MoT-2B**：2B activated parameters，4B total parameters，主打 edge deployment（边缘部署）。
- **HY-Embodied-0.5 MoE-A32B**：32B activated parameters，407B total parameters，主打 complex reasoning（复杂推理）。

摘要里最重要的 claim 是三条：

1. MoT-2B 在 22 个 benchmark 中相对同规模模型表现强，16 个任务第一。
2. A32B 可以达到接近 frontier models 的水平，平均分超过 Gemini 3.0 Pro。
3. 在 downstream robot control 中，使用 HY-Embodied-0.5 训练 VLA 能在真实任务上取得有竞争力的成功率。

这里要注意：论文主贡献并不只是 VLA 控制，而是“先训练一个 embodied VLM brain，再把它接到 VLA action expert”。用户提到的 `Hy-Embodied-0.5-VLA` 对应的主要是第 6 节机器人控制实验，而 arXiv 条目本身是 HY-Embodied-0.5 tech report。

## 1. Introduction / 为什么需要具身基础模型

Introduction 从 intelligent agents（智能体）的扩展开始：LLM agents 已经能做 coding、personal assistance、workflow automation、scientific research 等数字世界任务，但进入 physical environments（物理环境）后，VLM 仍不够。作者把差距拆成两个主要维度。

第一是 **Fine-Grained Visual Perception（细粒度视觉感知）**。机器人要抓、放、插、叠、挂，不能只知道“图里有杯子”，还要知道物体边界、深度、3D 位置、可交互部位、与夹爪之间的空间关系。通用 VLM 在 web-scale static data 上训练，常缺这种 physical grounding（物理落地）能力。

第二是 **Embodied Prediction, Interaction, and Planning（具身预测、交互与规划）**。真实操作不是静态问答，而是动态过程。模型要判断下一步该做什么、动作后状态会怎样变化、当前步骤是否完成、哪个对象可供抓取或移动。这类能力需要 trajectory、affordance、long-horizon planning 和 future-state reasoning 数据。

作者提出 HY-Embodied-0.5 的出发点是：embodied VLMs 可以桥接 LLM agents 和 physical agents。也就是说，VLM 不一定直接输出关节动作，但可以作为机器人系统的 cognitive core（认知核心），提供视觉-语言-空间推理，再交给 action module。

Introduction 中还概述了模型族：

- MoT-2B：2B activated / 4B total parameters，追求 real-time responsiveness 和 edge deployment。
- MoE-A32B：32B activated / 407B total parameters，追求复杂视觉感知和具身推理。

最后，作者给出评估主线：22 个 public benchmarks 覆盖 visual perception、spatial reasoning、embodied understanding；MoT-2B 平均 58.0%，比 Qwen3-VL-4B 和 RoboBrain2.5-4B 高；A32B 平均 67.0%，超过 Gemini 3.0 Pro 的 63.6；真实 robot control 展示模型可迁移到 VLA。

## 2. Model Architecture / 模型架构

![[papers/images/2604.07430v1/2-arch_page1.png|700]]

**Fig. 2: HY-Embodied-0.5 Mixture-of-Transformers Architecture.** 图里最重要的是视觉和语言 token 的计算分开：vision branch 有 Vision MoT、Local Full Attention 和 Vision Loss；language branch 有 Language MoT、Global Causal Attention 和 LLM Loss；视觉输入末尾还有 Visual Latent token，用 Global Loss 监督。

### 2.1 HY-ViT 2.0: Efficient Native-Resolution Visual Encoder

HY-ViT 2.0 是视觉编码器。它的任务是把 visual inputs（视觉输入）投影到 language embedding space，让 LLM 可以处理视觉和文本。作者强调三点：

- 支持 arbitrary-resolution / native-resolution input（任意或原生分辨率输入），对机器人所需细粒度视觉线索很重要。
- 使用更大规模预训练数据，并引入 tiny LLM 提供 language supervision signals。
- 加入 visual reconstruction supervision，减少视觉信号进入 LLM 前的信息损失。

MoT-2B 使用 400M 参数的 ViT，并从更强内部 ViT distill（蒸馏）知识。作者还训练一个更大的 ViT 生成 discrete visual representations（离散视觉表示），codebook size 是 2k，每个 $8\times 8$ image patch 压成一个 discrete code。这些 code 后面用于 vision branch 的 next-code prediction supervision。

直观理解：HY-ViT 2.0 不是只做 CLIP-style global image embedding，而是尽量保留局部视觉细节。对 embodied tasks 来说，局部细节通常决定物体是否能抓、距离多远、边界在哪里。

### 2.2 Modality-Adaptive Computing with Mixture-of-Transformers

MoT 的核心是 modality-adaptive computing（模态自适应计算）。具体做法是：在多模态训练开始前，复制 language model 的 Feed-Forward Network (FFN) 和 QKV 参数。forward 时：

- visual tokens 使用 duplicated vision-specific parameters；
- text tokens 使用 original text-specific parameters。

这样做的理由是：heavy visual training 往往会拉低原有语言能力，而小模型容量有限，视觉和文本如果完全共享参数，冲突更明显。MoT 相当于给视觉 token 增加非共享容量，同时保持推理开销接近 dense 2B baseline。

![[papers/images/2604.07430v1/3-mask_page1.png|620]]

**Fig. 3: Attention Computation of Modality-Adaptive MoT.** 视觉 token 没有语言序列那种天然单向性，所以作者给 visual tokens 用 bidirectional / full attention，而 text tokens 仍用 causal attention。对于 interleaved multi-modal sequences（图文交错序列），不同 token 类型采用不同 attention mask。

作者还给 vision branch 加一个 visual next-code prediction task。设 teacher ViT 为第 $i$ 个视觉 token 生成目标 discrete code $z_i$，模型预测概率分布为 $p_i$，视觉 token 数为 $N_v$，vision loss 为：

$$
\mathcal{L}_{vision}
=-\frac{1}{N_v}\sum_{i=1}^{N_v}\log p_i(z_i).
$$

直观含义：视觉分支不只服务最终文本答案，还要学会预测视觉 patch 的离散语义/重构 code，因此更重视局部视觉建模。

### 2.3 Visual Latent Tokens Connecting Vision and Language

Visual latent tokens 是每个 visual element（图像或视频帧）末尾附加的 learnable token。作者用 teacher ViT 的 global CLS feature 监督它，使它承载整幅图的全局视觉语义。

设 latent token 的 hidden state 经映射后为 $f_{latent}$，teacher ViT 的 global CLS feature 为 $f_{teacher}$，global loss 是 negative cosine similarity：

$$
\mathcal{L}_{global}
=-
\frac{f_{latent}^{\top} f_{teacher}}
{\|f_{latent}\|\|f_{teacher}\|}.
$$

预训练总目标是：

$$
\mathcal{L}_{total}
=\mathcal{L}_{llm}
+\mathcal{L}_{vision}
+\mathcal{L}_{global}.
$$

到 mid-training 和 fine-tuning 阶段，作者丢掉 vision/global supervision，只优化 autoregressive language loss $\mathcal{L}_{llm}$。

## 3. Pre-training / 预训练数据和训练策略

![[papers/images/2604.07430v1/4-pretrain_page1.png|700]]

**Fig. 4: Data Distribution for Pre-training and Mid-training Stages.** 图里有两个关键规模：pre-training 阶段超过 200B embodied/perception tokens，mid-training 阶段超过 12M high-quality QA pairs。正文进一步给出完整 pre-training mixture 超过 600B tokens。

### 3.1 Pre-training Data / 数据构成

作者把数据分为 visual perception data、embodied-centric data、spatial-centric data 和 general understanding data。

#### 3.1.1 Visual Perception Data

**Omni-Detection.** 这部分有 62M samples，来源包括 OpenImages、Objects365、RefCOCO、SA-1B 等。任务包括 object tagging、2D bounding boxes 和 3D bounding boxes。对于无标注或低质量标注数据，作者用 VLM 识别对象，再结合 SAM 和 VLM grounding 生成坐标，并用更强 VLM teacher 验证。坐标统一归一化为 0 到 1000 的整数格式。

**Depth Estimation.** 这部分约 36M samples，覆盖 absolute depth（绝对深度）和 relative depth（相对深度）。作者从室内/室外 3D 数据集和自动驾驶数据中构造基于特定像素坐标的 QA。数据清洗会排除 object boundaries、infinity pixels 和物理不一致区域，并归一化 camera focal lengths 以统一尺度。

**Segmentation.** 这部分约 5M samples，来自 SA-1B 的高质量 segmentation maps。作者过滤过小、过大、碎片化 masks，并参考 PaliGemma 把 binary masks 编码进 tokenizer vocabulary，形成结构化 QA。

**Pointing and Counting.** 这部分约 11M samples。pointing（指点）和 counting（计数）对 VLM 很难，因为容易 spatial hallucination（空间幻觉）和枚举错误。机器人操作却需要精确 point-level perception。因此作者从 Pixmo-Points 等数据和高密度目标场景中构造难度更高的指点/计数数据。

#### 3.1.2 Embodied-Centric Data

Embodied-centric data 面向 physical agents 的操作需求，被组织成三层：

- embodied perception（具身感知）：建立空间和物理意识；
- semantic understanding（语义理解）：把视觉输入和上下文推理连接起来；
- high-level planning and manipulation（高层规划与操作）：监督顺序决策和动作执行。

具体类别如下：

| 类别 | 数据/任务 | 作用 |
| --- | --- | --- |
| Grounding | Molmo、RoboPoint、RefSpatial、自有标注；点定位、bbox、referring expression | 让模型定位交互对象和机器人 manipulator。 |
| Affordance | RoboAfford、ShareRobot，以及生成的操作指令 | 判断用户指令中对象哪里可操作。 |
| Trajectory | MolmoAct、ShareRobot、FSD，以及机器人视频中用 CoTracker3 抽取 arm/agent motion | 训练模型预测 waypoint-level 运动趋势。 |
| Understanding | Robo2VLM、RoboVQA、RoboRefit、RoboInter-VQA 等 | 训练任务状态、空间关系和图中 annotation 理解。 |
| Planning | 机器人操作视频自动标注任务、时序切片成后续动作 QA | 训练模型根据已完成步骤预测后续 action sequence。 |
| Reasoning | 内部 long-horizon reasoning 数据 | 覆盖 action sequencing、multi-image action comprehension、future state prediction、visual puzzles、intuitive physics。 |

Trajectory 数据有一个重要细节：作者从 manipulation videos 中抽 arm/agent 轨迹，最多 downsample 成 15 waypoints，并画到图像上，再用更强 VLM judge 过滤准确性。这个设计让模型学习“视觉场景中的动作路径”而不只是文本规划。

#### 3.1.3 Spatial-Centric Data

Spatial-centric data 关注从视觉观察理解 3D 环境，不一定直接涉及机器人动作。作者分成五类，来源包括 ScanNet、ScanNet++、ARKitScenes 和自采数据。

| 类别 | 训练内容 | 为什么对具身重要 |
| --- | --- | --- |
| Correspondence | cross-frame point matching、2D-3D instance mapping | 多视角理解、物体跟踪和空间对齐的基础。 |
| Geometry | point depth estimation、depth comparison | 让模型知道远近、尺度和深度关系。 |
| Configuration | object counting、relative distance、relative direction、distance ranking | 训练“谁在谁左边/前面/最近”等空间关系。 |
| Measurement | object size、object-object distance、room floor area | 训练公制度量估计，服务真实物理空间。 |
| Dynamics | camera ego-motion、object movement | 训练时序变化、运动方向和动态场景理解。 |

这里最值得回看的是 Measurement 和 Dynamics。很多 VLM 可以回答“哪个物体在左边”，但机器人还需要厘米、米、地面面积、移动方向等物理量。HY-Embodied-0.5 显式把这些做成 QA 训练任务。

#### 3.1.4 General Understanding Data

作者没有抛弃通用能力，而是加入 in-house general VLM data，覆盖 image captioning、world knowledge、STEM、math、coding、scientific reasoning、document/chart/OCR、多轮对话、多图上下文、complex instruction following、GUI navigation 等。这是为了避免模型变成只会机器人 QA 的窄模型。

### 3.2 Training Recipe / 训练配方

![[papers/images/2604.07430v1/5-pipeline_page1.png|700]]

**Fig. 5: Training Pipeline for HY-Embodied-0.5 Series.** 这张图展示从 Hunyuan-1.8B / HY-ViT-2.0 到 MoT-2B，再到 embodied post-train、self-evolution 和 on-policy distillation 的流程；A32B 路径则通过 short-chain embodied SFT、thinking cold start、GRPO RL 和 RFT 得到强教师。

训练分两阶段：

| 阶段 | 规模 | 主要设置 |
| --- | --- | --- |
| Pre-training | 超过 600B tokens，其中 389B general understanding、236B embodied/perception | base LR $5\times10^{-5}$，ViT LR $5\times10^{-6}$，weight decay $10^{-4}$，global batch 256，context 32k；ViT、MoT、visual latent tokens 都可训练。 |

### 3.3 Training Strategy / 多损失策略

训练目标有三项：

- $\mathcal{L}_{llm}$：标准 autoregressive language loss，训练文本/答案生成。
- $\mathcal{L}_{vision}$：视觉 next-code prediction，监督 vision branch。
- $\mathcal{L}_{global}$：visual latent token 和 teacher ViT global feature 的对齐损失。

总损失：

$$
\mathcal{L}_{total}
=\mathcal{L}_{llm}
+\mathcal{L}_{vision}
+\mathcal{L}_{global}.
$$

直观上，$\mathcal{L}_{llm}$ 保证模型会回答，$\mathcal{L}_{vision}$ 保证局部视觉 token 不丢信息，$\mathcal{L}_{global}$ 保证 visual latent token 带全局语义。这个组合正对应本文的目标：既要看得细，也要能推理。

## 4. Post-training / 后训练

后训练是这篇的另一个核心。作者不是简单做一次 instruction tuning，而是用 SFT、RL、RFT 和 OPD 组合出一个 self-evolving pipeline。

### 4.1 Supervised Fine-tuning / 冷启动 CoT

SFT 阶段聚焦 long-chain reasoning。作者从 spatial、embodied、general 数据源中抽 high-complexity multi-step problems，再通过 human-model collaborative pipeline 构造 Chain-of-Thought trajectories。这些 CoT 会由 LLM 从 reasoning quality、logical correctness、sequence repetition 等维度评估，同时检查 final answer exact match。最终得到约 100k cold-start CoT instances。

训练细节：仍用 cross-entropy loss，但不做 sequence packing，每个样本独立处理，以突出完整推理链；base learning rate 保持 $5\times10^{-5}$。

### 4.2 Reinforcement Learning / 具身 RL

![[papers/images/2604.07430v1/6-rl_page1.png|700]]

**Fig. 6: Reward Designs for Embodied Reinforcement Learning.** 奖励被分为 Grounding-Based、Regression-Based、Trajectory-Based 和 Textual-Based 四类。图里的核心思想是：不同输出结构需要不同 reward，不能用一个 exact match 解决所有具身任务。

#### Data Construction / 动态选 frontier samples

RL 数据不是固定集合。每轮用当前模型在大候选池上 multi-sample evaluation：

- 每次都答对的样本太简单，丢弃。
- 每次都答错的样本太难，丢弃。
- 只在部分 rollouts 中成功的样本保留，因为它们最靠近当前 capability frontier（能力前沿）。

每个 RL stage 使用新构造的 50K samples，并在 perception、prediction、interaction、planning 等能力之间平衡。这是一个 capability-adaptive curriculum（能力自适应课程）。

#### Reward Designs / 结构化奖励

作者把 reward 写成：

$$
r = R_t(y,y^\star)\in[0,1].
$$

其中 $y$ 是模型输出，$y^\star$ 是参考答案，$R_t$ 是按任务类型选择的 reward function。

具体包括：

| 输出类型 | 奖励方式 | 为什么这样设计 |
| --- | --- | --- |
| grounding / localization | IoU、Hungarian-matched IoU、normalized point distance、Chamfer distance | 空间位置有部分正确性，不能只 exact match。 |
| trajectory / planning path | DTW、Fréchet distance、endpoint consistency | 路径相似度比逐 token 匹配更合理。 |
| discrete / constrained outputs | exact match、mixed matching、LCS-style partial credit | 多选、二元判断、排序等有结构化答案。 |
| continuous estimation | relative-error decay reward | 数值估计需要平滑惩罚。 |
| free-form reasoning | LLM-as-a-Judge | 无法可靠 parse 时才用 judge。 |

开放式任务的 judge reward：

$$
r_{\text{free}} = J(q,y,y^\star).
$$

这里 $q$ 是输入问题，$J$ 是 LLM judge。作者强调：能用 deterministic / structure-aware reward 时优先不用 LLM judge。

#### GRPO Objective / 强化学习目标

给定 multimodal input $x=(I,Q)$，从旧策略 $\pi_{\theta_{old}}$ 采样 $G$ 个 responses，得到 rewards $\{r_1,\ldots,r_G\}$。组内相对优势为：

$$
A_i=
\frac{r_i-\mu(\mathbf{r})}{\sigma(\mathbf{r})},
\qquad
\mathbf{r}=\{r_1,\dots,r_G\}.
$$

策略比率：

$$
\rho_{i,t}
=
\frac{\pi_{\theta}(y_{i,t}\mid x,y_{i,<t})}
{\pi_{\theta_{old}}(y_{i,t}\mid x,y_{i,<t})}.
$$

RL loss：

$$
\mathcal{L}_{\mathrm{RL}}(x)
=
-\frac{1}{\sum_{i=1}^{G}|y_i|}
\sum_{i=1}^{G}\sum_{t=1}^{|y_i|}
\min\left(
\rho_{i,t}A_i,\,
\mathrm{clip}(\rho_{i,t},1-\epsilon_{\mathrm{low}},1+\epsilon_{\mathrm{high}})A_i
\right).
$$

训练设置包括：group size $G=16$，最大 prompt 16,384 tokens，最大 response 16,384 tokens，temperature 1.0，top-p 1.0，top-k -1，batch size 128，learning rate $8\times10^{-7}$，每个 RL stage 5 epochs。clip ratio 使用非对称范围 $[0.8,1.35]$，作者认为长链多模态 RL 中更稳定。

### 4.3 Evolving Deep Thinking with Iterative Training / RL 与 RFT 交替

RL 能通过 reward search 找到更好行为，但不保证 reasoning traces（推理轨迹）质量稳定。作者因此引入 RFT：

1. 从 RL 后最新 checkpoint 开始，在 curated data pool 上多采样。
2. 用和 reward 一致的准则离线评估结果。
3. 保留部分成功的样本，丢弃已饱和或完全不可达样本。
4. 用 stronger teacher model 评价 thinking quality。
5. 从约 1M candidates 中过滤出约 300K high-quality traces，再做 SFT。

直观理解：RL 负责扩大 capability frontier，RFT 把偶然发现的好推理过程固化成稳定行为。作者交替进行 RL 和 RFT，让模型逐步 self-evolve。

### 4.4 Large-to-Small On-Policy Distillation / 大到小在策略蒸馏

目标是把大模型的 embodied reasoning ability 转给 compact MoT-2B。作者认为传统 offline distillation 不够，因为学生只看 teacher-generated trajectories，不会在自己生成的错误前缀上得到教师监督。

OPD 的做法是：学生先自己 rollout：

$$
y=(y_1,\dots,y_T)\sim\pi_s(\cdot\mid x).
$$

然后教师在学生生成的 prefixes 上 teacher forcing。设教师和学生在 step $t$ 的 next-token distributions 为：

$$
\pi_t(\cdot\mid x,y_{<t}),\qquad
\pi_s(\cdot\mid x,y_{<t}).
$$

优化 KL：

$$
\mathcal{L}_{\mathrm{OPD}}
=
\mathbb{E}_{x,\,y\sim\pi_s(\cdot\mid x)}
\left[
\frac{1}{|y|}
\sum_{t=1}^{|y|}
\mathrm{KL}
\left(
\pi_t(\cdot\mid x,y_{<t})
\,\|\,\pi_s(\cdot\mid x,y_{<t})
\right)
\right].
$$

直观含义：学生在哪些状态会犯错，教师就在哪些状态上给分布监督。这比只模仿教师最终答案更接近部署时的真实分布。

## 6. Robot Control Results / VLA 机器人控制结果

这一节最贴近 `HY-Embodied-0.5 VLA`。作者不是直接让 VLM 输出动作，而是在 MoT-2B base model 上扩展 Action Expert module，结构参照 $\pi_0/\pi_{0.5}$，得到用于真实机器人控制的 Vision-Language-Action model。

训练流程：

1. **UMI pre-finetuning**：先用 5K hours UMI data fine-tune network。作者强调这些数据都来自 UMI，因此网络在这一阶段没有接触具体的目标机器人 embodiment。
2. **Real-robot SFT**：再对三个真实任务收集 300-700 episodes demonstrations，做 supervised fine-tuning。
3. **Fair baseline**：$\pi_0$ 和 $\pi_{0.5}$ 使用同样真实机器人数据量和训练迭代进行 SFT。

训练细节：UMI 阶段 per-GPU batch size 为 32，使用 32 GPUs，总共 200K iterations。

![[papers/images/2604.07430v1/robot_example.png|700]]

**Robot experimental setup.** 平台是 dual-arm Xtrainer，使用 head-mounted 和 wrist-mounted cameras。任务包括 Precision Plug-in Packing、Tableware Stacking、Mug Hanging。每个任务每个模型 20 次 real-robot trials，object poses 每次随机初始化。

## 7. Conclusion / 结论

结论重申 HY-Embodied-0.5 是 strong foundation VLM for real-world embodied tasks。作者认为它的关键是两点：

- modality-adaptive Mixture-of-Transformers + visual latent tokens 提供 fine-grained spatial and visual perception；
- embodied post-training pipeline 把 deep reasoning capabilities 压进 2B activated edge variant。

最后一句很重要：作者计划继续 bridge the gap between language and action models，训练更适合 complex real-world applications 的 real-world brain。也就是说，HY-Embodied-0.5 现在仍主要是“VLM brain”，第 6 节只是展示它可以接到 Action Expert 形成 VLA。后续真正的问题是：语言-视觉推理和低层动作模型之间如何更紧密闭环。

## 公式与变量速查

### Vision Loss

$$
\mathcal{L}_{vision}
=-\frac{1}{N_v}\sum_{i=1}^{N_v}\log p_i(z_i).
$$

- $N_v$：visual tokens 数量。
- $p_i$：模型对第 $i$ 个视觉 token 的 code 概率分布。
- $z_i$：teacher ViT 生成的目标 discrete code。
- 用途：监督 MoT vision branch 学局部视觉重构/语义。

### Global Loss

$$
\mathcal{L}_{global}
=-
\frac{f_{latent}^{\top}f_{teacher}}
{\|f_{latent}\|\|f_{teacher}\|}.
$$

- $f_{latent}$：visual latent token 的映射后 hidden state。
- $f_{teacher}$：teacher ViT 的 global CLS feature。
- 用途：让 latent token 承载整幅图的全局视觉语义。

### Total Pre-training Loss

$$
\mathcal{L}_{total}
=\mathcal{L}_{llm}
+\mathcal{L}_{vision}
+\mathcal{L}_{global}.
$$

- $\mathcal{L}_{llm}$：标准语言建模损失。
- 用途：同时训练语言输出、局部视觉和全局视觉桥接。

### Task-Aware Reward

$$
r=R_t(y,y^\star)\in[0,1].
$$

- $R_t$：按任务类型选择的 reward function。
- $y$：模型输出。
- $y^\star$：参考答案。
- 用途：为 grounding、trajectory、regression、textual reasoning 等异构任务提供不同奖励。

### Free-Form Judge Reward

$$
r_{\text{free}}=J(q,y,y^\star).
$$

- $J$：LLM judge。
- $q$：输入问题。
- 用途：规则无法可靠判断的开放式具身推理任务。

### GRPO Advantage

$$
A_i=
\frac{r_i-\mu(\mathbf{r})}{\sigma(\mathbf{r})},
\qquad
\mathbf{r}=\{r_1,\dots,r_G\}.
$$

- $G=16$：每个输入采样的 response group size。
- 用途：在异构 reward 尺度下做组内相对归一化。

### OPD Loss

$$
\mathcal{L}_{\mathrm{OPD}}
=
\mathbb{E}_{x,\,y\sim\pi_s(\cdot\mid x)}
\left[
\frac{1}{|y|}
\sum_t
\mathrm{KL}
\left(
\pi_t(\cdot\mid x,y_{<t})
\,\|\,\pi_s(\cdot\mid x,y_{<t})
\right)
\right].
$$

- $\pi_s$：student model。
- $\pi_t$：teacher model。
- $y\sim\pi_s$：学生自己 rollout 出来的响应。
- 用途：教师在学生真实会走到的前缀上监督学生，减少 train-inference mismatch。

## 局限与可追问点

### 1. VLA 控制实验规模偏小

第 6 节只有 3 个真实任务，每项 20 trials。结果很有价值，但不能直接推出 HY-Embodied-0.5 对所有 VLA 控制都优于 $\pi_0/\pi_{0.5}$。尤其 Tableware Stacking 中 HY 仍略低于 $\pi_{0.5}$。

可追问：如果在 LIBERO、BridgeData、DROID、BEHAVIOR-1K 等更大控制 benchmark 上系统评测，会怎样？VLA 的失败模式是视觉理解错、动作专家错，还是数据不足？

### 2. 训练数据和内部教师模型不可完全复现

论文列出数据规模和类别，但大量 in-house data、teacher VLM、annotation filtering、reasoning data 和 robot data 细节没有完全公开。开源模型可用，但完整训练 pipeline 难以复现。

可追问：公开模型的能力主要来自数据规模、数据质量、MoT 架构、后训练，还是内部 teacher 的筛选能力？

### 3. CoT 和真实行动之间的关系还需验证

论文展示 long-chain CoT 很强，但真实机器人控制未必总需要长文本推理。对于快速闭环操作，过长 reasoning 可能带来延迟或不稳定。

可追问：在 action expert 中，CoT 表示是否真的被使用？thinking mode 对 VLA success rate 有无消融？短链/无链控制是否更稳定？

### 4. 空间/具身 benchmark 和物理成功率之间仍有 gap

22 benchmarks 多数是 VLM-style QA、planning、spatial reasoning 或 trajectory prediction。它们衡量 cognition，但不等同于闭环控制、接触动力学和错误恢复。

可追问：哪些 benchmark 分数最能预测真实 robot success？DA-2K、MindCube、RoboBench、ShareRobot-Traj 与 VLA 任务之间是否有相关性？

### 5. Edge deployment 仍有硬件门槛

MoT-2B 是 2B activated，但总参数 4B，Hugging Face 权重约 7.6GB。对机器人本体边缘计算来说仍需要合适 GPU/NPU 和低延迟推理优化。

可追问：在真实机器人部署中，VLM backbone 推理频率是多少？是否在线 thinking？是否能在低功耗硬件上实时运行？

### 6. Safety and physical constraints 没有成为主线

论文主要优化能力和 benchmark 分数，较少讨论安全约束、碰撞、力控、失败恢复、异常检测。真实机器人系统中这些通常比 QA benchmark 更关键。

可追问：HY-Embodied-0.5 能否输出 uncertainty、risk assessment 或 safety-aware plans？能否和 motion planner / force controller 组合？

## 为什么需要回看这篇

这篇适合作为 “embodied VLM foundation for VLA” 的核心入口。以后看以下方向时可以回看：

- **VLA backbone**：比较 $\pi_0/\pi_{0.5}$、OpenVLA、RoboBrain、MiMo-Embodied 和 HY-Embodied-0.5 时，理解具身 VLM 基座如何训练。
- **VLA 数据工程**：它把 detection、depth、segmentation、pointing/counting、trajectory、planning、spatial measurement 都纳入训练，是数据配方参考。
- **小模型具身推理**：MoT-2B 说明 2B activated 模型也可以通过数据、MoT 和 distillation 做到较强 embodied/spatial reasoning。
- **语言-动作桥接**：第 6 节展示从 embodied VLM 到 Action Expert 的接口，可和 FRS 的 policy steering、SERF 的 map tokens 放在一起比较。
- **真实机器人验证标准**：它的 3-task real-world eval 很初步，但提供了一个“VLM foundation -> UMI pretraining -> task SFT -> real robot trials”的链条。
